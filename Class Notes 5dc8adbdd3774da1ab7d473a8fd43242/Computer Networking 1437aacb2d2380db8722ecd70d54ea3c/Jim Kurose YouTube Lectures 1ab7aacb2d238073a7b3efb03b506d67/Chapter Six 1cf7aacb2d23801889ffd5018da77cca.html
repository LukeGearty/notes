Error Detection and Correction

EDC: error detection and correction bits

D: Data protected by error checking, may include header fields

Datagram sends it down, D and EDC goes through a bit error prone link (can introduce bit errors) and the receiver does a check to see if bit errors were introduced. if they were, then it drops, if it doesn’t, it sends it up the protocol stack.

Parity Checking

Single bit parity: detect single bit errors

A parity bit is sent to 0 or 1. if 1, there is an even number of 1’s. the receiver checks if there is an even number of 1 error bits. Computes parity of d + 1 received bits, if not even then error detected. Can detect odd number of bit flips.

Two-Dimensional Parity:

computing a parity bit for each row and column. There is better protection, so it can detect two-bit errors.

In the case of a single bit error, it can detect and correct single bit errors without retransmission.

Internet Checksum

Goal: Detect errors (ie flipped bits) in transmitted segment

sender: treat contents fo UDP segment as sequence of 16-bit integers.

Checksum: addition of segment content, 1’s complement sum

Checksum value put into UDP checksum field

Receiver: compute checksum of received segment

check if computed checksum equals checksum field value

Not equal - error detected

Equal - no error detected, may be errors

Cyclic Redundancy Check (CRC)

more powerful error-detection coding

D: data bits (given, think of these as a binary number)

G: bit pattern (generator), of r+1 bits (given, specified in CRC standard)

sender: compute r CRC bits, R such that <D,R> exactly divisible by G (mod 2)

Receiver knows G, divides <D,R> by G. If non-zero remainder: error detected

Can detect all burst errors less than r+1 bits

Widely used in practice - Ethernet, 802.11 WiFi

Sender wants to compute R

such that:

D * 2^r = nG XOR R

which says:

if we divide D * 2^r by G, we want remainder R to satisfy

R = Remainder [ (D* 2^r) / G ]
