Section 4.1

R1.  Let’s review some of the terminology used in this textbook. Recall that the name of a transport-layer packet is segment and that the name of a link-layer packet is frame. What is the name of a network-layer packet? Recall that both routers and link-layer switches are called packet switches. What is the fundamental difference between a router and link-layer switch?

A network-layer packet is called a datagram. The fundamental difference between a router and a switch is that routers are used to communicate between networks, while switches are used within networks. Routers use IP addresses, link-layer switches use MAC addresses.



R2.  We noted that network layer functionality can be broadly divided into data plane functionality and control plane functionality. What are the main functions of the data plane? Of the control plane?

Data Plane: Handles the local action of forwarding packets from input ports to output ports

Control Plane: handles the global action of routing.

R3.  We made a distinction between the forwarding function and the routing function performed in the network layer. What are the key differences between routing and forwarding?

The key distinctions between forwarding and routing is that forwarding is a local action concerned with getting the packet from input to output, while routing is a global action concerned from getting the packet from source to destination.

R4.  What is the role of the forwarding table within a router?

The forwarding table is used to determine which packets go out which output ports.

R5.  We said that a network layer’s service model “defines the characteristics of end-to-end transport of packets between sending and receiving hosts.” What is the service model of the Internet’s network layer? What guarantees are made by the Internet’s service model regarding the host-to-host delivery of datagrams?

The service model of the Internet’s Network Layer is best-effort service - it makes the best attempt, but there are no guarantees regarding delivery, bandwidth, security, etc.

Section 4.2

R6. In Section 4.2, we saw that a router typically consists of input ports, output ports, a switching fabric and a routing processor. Which of these are implemented in hardware and which are implemented in software? Why? Returning to the notion of the network layer’s data plane and control plane, which are implemented in hardware and which are implemented in software? Why?

Input Ports, Output Ports, switching fabric are implemented in hardware because they need to be fast - faster than software. Routing processor is implemented in software.

The data plane - responsible for local action, is implemented in hardware

Control plane - responsible for routing, is implemented in software

R7.  Discuss why each input port in a high-speed router stores a shadow copy of
the forwarding table.

So forwarding decisions can be made locally at each input port without invoking the centralized routing processor on a per-packet basis. This avoids central processing bottlenecks.

R8.  What is meant by destination-based forwarding? How does this differ from generalized forwarding (assuming you’ve read Section 4.4, which of the two approaches are adopted by Software-Defined Networking)

Destination-based forwarding is the destination of the packet determines the output port the packet will be sent on. Generalized forwarding is when the “match plus action” paradigm can include things like dropping packets, sending them out on specific ports, rewriting source IP and source port for NAT, etc. Software-Defined Networking adopts generalized forwarding.

R9. Suppose that an arriving packet matches two or more entries in a router’s forwarding table. With traditional destination-based forwarding, what rule does a router apply to determine which of these rules should be applied to determine the output port to which the arriving packet should be switched?

With traditional destination-based forwarding, the rule applied to determine which output port would be longest-prefix match.

R10.  Three types of switching fabrics are discussed in Section 4.2. List and briefly
describe each type. Which, if any, can send multiple packets across the fabric
in parallel?

Switching via memory: The router’s memory is used to store packets and forward them to the appropriate output port.

Switching via bus: A shared bus moves the packets from the input ports to the output ports.

Switching via interconnection network: a sophisticated network connects input ports to output ports. This can send multiple packets across the fabric in parallel, as long as they are going to different ports.



R11.  Describe how packet loss can occur at input ports. Describe how packet loss
at input ports can be eliminated (without using infinite buffers).


If the switch fabric is not fast enough to transfer all ariving packets through the fabric, then packet queuing can occur at input ports. It can also occur if packets are going to be moved to the same output ports. To eliminate packet loss at input ports, without using infinite buffers, the switching fabric needs to at least N times as fast as the input line speed, where N is the number of ports.

R12.  Describe how packet loss can occur at output ports. Can this loss be prevented by increasing the switch fabric speed?

If the switching fabric is N times faster than the input line speed, and N new packets arrive and are destined for the same output port, the N arriving packets will have to queue for transmission. Increasing the switching fabric speed would not affect the loss.

R13.  What is HOL blocking? Does it occur in input ports or output ports?

When processing one packet blocks other packets from being processed. The switching fabric is currently processing one port, so a packet has to wait even if it’s output port is free - it occurs in input ports.

R15.  Give an example showing why a network operator might want one class of
packets to be given priority over another class of packets.

Prioritizing well-known ports and their associated protocols over each other. For example, an SNMP network management datagram on Port 161 might be assigned a higher priority than an IMAP email protocol on port 143 or 993.



R16.  What is an essential different between RR and WFQ packet scheduling? Is
there a case (
Hint: Consider the WFQ weights) where RR and WFQ will
behave exactly the same?

The difference is that in WFQ, there are classes of packets like in RR, but each class in WFQ receives a weight based on a formula. If the formula leads to all classes having equal weight, then it would behave like RR.

R27.  What is meant by the term “route aggregation”? Why is it useful for a router
to perform route aggregation?

The ability to use a single prefix to advertise multiple networks. Suppose that an ISP is providing the address block 200.23.16.0 /20 to eight organizations. The ISP can advertise to the outside world that it should be sent any datagrams whose first 20 address bits match 200.23.16.0 / 20 . The rest of the world doesn’t need to be aware that within that address block there are 8 organizations.

This is useful because it can help reduce the number of routes used to access the network, the size of routing tables.

R28.  What is meant by a “plug-and-play” or “zeroconf” protocol?

A plug-and-play protocol or zero conf allows the automation of network related aspects of the protocol, without having to do things manually.



R29.  What is a private network address? Should a datagram with a private network
address ever be present in the larger public Internet? Explain.


A private network address is a network address whose address only has meaning to other devices within that network. Since the IP address wouldn’t have meaning to anyone outside of the network, it shouldn’t be present in the larger public Internet.

Section 4.4



R32.  How does generalized forwarding differ from destination-based forwarding?

Generalized forwarding follows the match plus action paradigm where the action can be multiple things like dropping a packet, NAT rewrite, etc. , whereas destination-based forwarding the action is just forwarding packets based on destination.



R33.  What is the difference between a forwarding table that we encountered in destination-based forwarding in Section 4.1 and OpenFlow’s flow table that we encountered in Section 4.4?

OpenFlow’s flow table provides other actions besides forwarding, like load balancing.



R34.  What is meant by the “match plus action” operation of a router or switch? In the case of destination-based forwarding packet switch, what is matched and what is the action taken? In the case of an SDN, name three fields that can be matched, and three actions that can be taken.

Match plus action is that a specific IP address range or protocol will have some kind of action for it, for example dropping packets.

In the case of an SDN, three fields can be IP SRC, IP DSt, TCP/UDP Src Port/Dst Port

Three actions can be forwarding, dropping, and modifying
