What is Network Security?

Alice and Bob want to communicate “securely”. Trudy is an intruder. What does that mean?

  • Alice wants only Bob to understand a message she has sent.
  • Bob wants to make sure the message from Alice is indeed from Alice.
  • Alice and Bob both want to make sure their messages were not altered in transit.
  • The both of them can indeed communicate in the first place.

There are a few properties of secure communication:

  • Confidentiality: Only the sender and intended recipient should be able to understand the messages. Eavesroppers may intercept the message, so this requires that the message be somehow encrypted.
  • Message integrity: The content of the messages are not altered in transit, either maliciously or accidentally. Extension to the checksumming techniques in rdt and data link protocols can provide message integrity.
  • End-point authentication. Sender and receiver should be able to confirm the identity of the other.
  • Operational Security: Since almost all organizations have networks that are attached to the internet, these networks therefore can potentially be compromised.

Communication can be intercepted, and potential attackers can eavesdrop - see messages between sender and receiver - or they can modify, insert, or delete messages between sender and receiver.

Principles of Cryptography

Cryptographic techniques allow a sender to disguise data so that an intruder gains no information from the intercepted data.

Plaintext: the message in its original form

Encryption Algorithm: Used to encrypt the plaintext. In modern techniques, the encryption technique is known to everyone, including the intruder

Ciphertext: the encrypted message

Key: A string of numbers or characters that is the input to the encryption algorithm. This is secret

The encryption algorithm takes the key and the plaintext message as input and produces ciphertext as output.

Decryption Algorithm: Takes a key and ciphertext as input and produces the original plaintext as output

Symmetric Key Systems: Both sender and receiver’s keys are identical

Public Key Systems: A pair of keys are used. One is known to both sender and receiver (and the entire world), the other key is known to either sender or receiver (not both).

Symmetric Key Cryptography

Caesar Cipher: Taking each letter in the plaintext message and substituting the letter that is k letters later (allowing wraparound).

Monoalphabetic Cipher: Rather than substituting the letter that is k letters later, any letter can be substituted for any other letter. For example, a→m, b→n, c→b, d→v, etc.

Problem:

E and T are the most frequently occuring letters in English, and two and three letter occurences of letters appear quite often (”in”, “it”, “the”, etc).

  • Ciphertext-only attack: The intruder has access only to the ciphertext w/ no info on the contents of the plaintext message
  • Known-plaintext: The attacker knows that some word or phrase is in the text. Ex: Enigma
  • Chosen-plaintext attack : the intruder is able to choose the plaintext message and obtain its corresponding ciphertext form.

Polyalphabetic Encryption: using multiple substitution ciphers. A single letter in the plaintext can be encrypted differently depending on where in the message it is

Block Ciphers

Used in PGP, TLS, and IPsec

The message is processed in blocks of k bits, and each block is encrypted independently.

Cipher-Block Chaining

If there are two plaintext blocks with the same phrase, a block cipher will produce the same output. For example, “HTTP/1.1” will have the same output, and this could clue attackers in on the encryption.

We can add randomness per block, but it is inefficient.

CBC adds one random value and chains encryptions between blocks.

Public Key Encryption

Since symmetric key encryption requires that the two communication parties share the symmetric key for encryption and decryption, that exchange by itself requires secrue communication.

Public Key is available to everyone in the world.

Private key is known only to sender and receiver.

Message Integrity and Digital Signatures

Message Integrity: messages originated from the correct sender and it was not tampered with.

Cryptographic Hash Functions

A hash function takes an input, m, and computes a fixed-size string H(m)known as a hash.

It is computationally infeasable to find any two different messages x and y such that H(x) = H(y).

Message Authentication Code

To perform message integrity between Alice and Bob, there must be an authentication key shared between them. Using this shared secret, message integrity can be performed:

1. Alice creates message m, concatenates s with m to create m + s, and calculates
the hash
H(m + s) (for example, with SHA-1). H(m + s) is called the message
authentication code (MAC)
.
2. Alice then appends the MAC to the message
m, creating an extended message
(
m, H(m + s)), and sends the extended message to Bob.
3. Bob receives an extended message (
m, h) and knowing s, calculates the MAC

H(m + s). If H(m + s) = h, Bob concludes that everything is fine.

Digital Signatures

A Digital signature is a cryptographic technique for achieving the goal of attesting that a person is the owner or creator of something.

Public Key Certification

Certifying that a public key belongs to aspecific entity.

End-Point Authentication

The process of proving your identity to another entity over a computer network.

Authentication is done as part of an authentication protocol, and is usually done solely on the basis of messages and data exchanged.

One approach is to authenticate based on IP address - easily exploitable.

A classic approach is to use a secret password. A flaw is that if someone is eavesdropping, then that password is compromised.

We can encrypt that password. But that leads to a new problem: The attacker doesn’t know the password, but can intercept the password and encrypted info and send it again.
