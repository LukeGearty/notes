Expressing Numbers:

A 64-bit machine can reason about 64 bits at a time.

Minimum: 0b0 = 0 = 0x0

What happens if you add 1 to 0xffffffffffffffff?

Integer Overflow = 0x10000000000000000

The extra bit gets put in common carry bit storage by the CPU and the result of the computation becomes 0

The inverse happens if we subtract 1 from 0

How to express negative numbers?
8 bit two’s complement

Take a number, 5 in binary: 0101

Flip the bits: 1010

and add one: 1011

The Need for Registers:
CPUs need to be fast, and CPUs need rapid access to data they’re working on. This is done via the Registers

Registers sit inside of the CPU. They are very fast, temporary stores for data.
For x86: eax, ecx, edx, ebx, esp, ebp, esi, edi

amd64: rax, rcx, rdx, rbx, rsp, rbp, rsi, rdi, r8, r9,r10,r11,r12,r13,r14,r15

Registers are typically the same size as the word width of the architecture ex: 64 bit architecture will have 64 bit registers


Partial Register Access:

Putting data to the register:

Ex:

Flow of data goes from right to left

Memory

Registers are expensive and we have a limited number of them. We need a place to store lots of data and have fairly fast access.

This place is called memory.

CPU has fastest access to registers.

Cache:

  • Cache is a small, fast memory located closer to the CPU than main memory (RAM). It stores frequently accessed data to reduce the time the CPU spends waiting to fetch data from slower memory.
    • L1 Cache: The fastest and smallest cache, located directly inside the CPU core.
    • L2 Cache: Larger but slower than L1, located near the CPU.
    • L3 Cache: Even larger and slower, shared across multiple CPU cores in modern processors.
  • Cache improves the overall performance by making sure the most needed data is quickly available to the CPU.

Each process memory is addressed linearly. From 0x10000 to 0x7ffffffff

It’s not all real, cause it is virtual!

Every C program consists of the Heap, Stack, Registers, and Instructions

The Heap is designed for manual memory allocation - in C, this is done by functions like malloc or calloc

Registers - small storage areas in processor that can store addresses, values, or anything that can be represented in 8 bytes or less

x86 architecture there are 6 General purpose registers:

eax, ebx, ecx, edx, esi, edi

Three registers for reserved:

ebp, esp, eip

The stack - a data structure that consists of two operations:

Push - puts an element at the top of the stack

Pop - removes the top element of the stack

Each element has a stack address - elements higher on the stack have a lower address than those lower on the stack

When a function is called, it is given a Stack Frame

EBP register: base pointer, contains address of the base of the current stack frame

ESP register: stack pointer, contains address of the top element of the current stack frame

Example:

#include <stdio.h>
#include <stdlib.h>

void func(int x) {
	int a = 0;
	int b = x;
}

int main() {
	func(10);
	
	return 0;
}

First, the value of the argument is pushed on the stack (10 in this case). Then, the return address of the function is pushed onto the stack. The return address is the 4 byte address of the instruction that will be executed as soon as the function has gone out of scope.

Then, the base pointer (EBP) is pushed on the stack. Then the stack pointer is given the address of the base pointer. The stack pointer is decremented to make room for the local variables. Everything between base pointer and stack pointer is the stack frame.

Since the first variable is an int with value of 0, the address of a is EBP - 4 bytes (an integer is 4 bytes)

For the second variable, the the functions argument is 8 bytes above the EBP. It cannot be moved directly to the stack frame. It is copied to a general purpose register. Then, it is moved 4 bytes below the first variable, 8 bytes below the base pointer.
