Static vs Dynamic, Strong vs Weak

Data Types: the type of data being dealt with, ex: string, number, int, boolean, float, double, etc.

Statically vs Dynamically Typed

Dynamic Type: You don’t have to specify types when writing the variable

ex: x = 5, x = “5”

Python, Javascript

Tradeoff: Things can be buggy

Statically Typed: You specify type when declaring the type, and cannot change it

ex: int x = 5

C, C++, Java

Fewer bugs


Strongly Typed Languages

  • Definition: Strongly typed languages enforce strict rules regarding type compatibility. Operations between mismatched types require explicit conversion.
  • Characteristics:
    • Prevent implicit type conversions that might lead to unexpected behavior.
    • Types must be explicitly declared or inferred, and mismatched types will cause compile-time or runtime errors.
    • Example:
      python
Copy code
x = 10  # Integer
y = "20"  # String
z = x + y  # Error: Cannot add integer and string
  • Examples: Python, Java, C#, Ruby.
  • Advantages:
    • Reduces errors caused by unintended type coercion.
    • Makes code more predictable and easier to debug.
  • Disadvantages:
    • Requires more explicit handling of types, which can slow down initial coding.

Weakly Typed Languages

  • Definition: Weakly typed languages allow operations between different types with automatic type conversions (type coercion).
  • Characteristics:
    • Implicit type conversions can occur, often leading to surprising results.
    • Developers have less control over how types interact.
    • Example:
      javascript
Copy code
let x = 10; // Integer
let y = "20"; // String
let z = x + y; // "1020" (Integer coerced to String)
  • Examples: JavaScript, PHP, Perl.
  • Advantages:
    • Simplifies coding by handling type conversions automatically.
    • Allows for quicker prototyping and flexibility.
  • Disadvantages:
    • Increased risk of bugs due to unintended type coercion.
    • Debugging may become harder as type errors can propagate unpredictably.
