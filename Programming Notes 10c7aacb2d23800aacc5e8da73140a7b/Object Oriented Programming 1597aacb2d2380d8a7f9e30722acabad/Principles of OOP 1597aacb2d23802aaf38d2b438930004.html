Principles of OOP

1. Encapsulation

Definition: Encapsulation is the bundling of data (attributes) and methods (functions) that operate on that data into a single unit (class). It restricts direct access to some of the object’s components, ensuring controlled interaction.

  • Key Features:
    • Access Modifiers: Use public, protected, and private (or their equivalents in Python) to control access.
    • Protects internal object state from unintended interference.
    • Encourages the use of getters and setters for controlled access.
  • Example in Python:
python
Copy code
class Account:
    def __init__(self, balance):
        self.__balance = balance  # Private attribute

    def deposit(self, amount):
        if amount > 0:
            self.__balance += amount

    def withdraw(self, amount):
        if amount > 0 and amount <= self.__balance:
            self.__balance -= amount

    def get_balance(self):
        return self.__balance

# Usage
acc = Account(100)
acc.deposit(50)
print(acc.get_balance())  # 150
acc.__balance = 1000  # Direct access fails due to encapsulation

2. Inheritance

Definition: Inheritance allows a class (child) to inherit attributes and methods from another class (parent). It promotes code reuse and establishes a natural hierarchy.

  • Key Features:
    • Parent class: The base or super class.
    • Child class: The derived or sub class.
    • Supports method overriding and extension of behavior.
  • Types of Inheritance:
    • Single Inheritance: One parent, one child.
    • Multiple Inheritance: Multiple parents for a single child.
    • Multilevel Inheritance: A chain of inheritance.
    • Hierarchical Inheritance: One parent, multiple children.
  • Example:
python
Copy code
class Animal:
    def speak(self):
        return "I make a sound"

class Dog(Animal):
    def speak(self):
        return "Bark"

# Usage
a = Animal()
d = Dog()
print(a.speak())  # I make a sound
print(d.speak())  # Bark

3. Polymorphism

Definition: Polymorphism means "many forms." It allows objects of different classes to be treated as objects of a common parent class, enabling flexibility and scalability.

  • Key Features:
    • Method Overriding: A child class provides a specific implementation for a method defined in its parent class.
    • Method Overloading (limited in Python): Achieved by defining methods with default arguments or by varying arguments.
    • Enables dynamic method invocation (e.g., duck typing in Python).
  • Example:
python
Copy code
class Animal:
    def speak(self):
        raise NotImplementedError("Subclasses must implement this method")

class Dog(Animal):
    def speak(self):
        return "Bark"

class Cat(Animal):
    def speak(self):
        return "Meow"

# Polymorphism
animals = [Dog(), Cat()]
for animal in animals:
    print(animal.speak())  # Outputs "Bark" and "Meow"

4. Abstraction

Definition: Abstraction hides implementation details and shows only the essential features of the object, enabling the user to interact with the object at a higher level.

  • Key Features:
    • Achieved using abstract classes and interfaces.
    • Focuses on what an object does rather than how it does it.
  • Benefits:
    • Simplifies complex systems by breaking them into manageable pieces.
    • Reduces code duplication.
  • Example:
python
Copy code
from abc import ABC, abstractmethod

class Shape(ABC):
    @abstractmethod
    def area(self):
        pass

    @abstractmethod
    def perimeter(self):
        pass

class Rectangle(Shape):
    def __init__(self, width, height):
        self.width = width
        self.height = height

    def area(self):
        return self.width * self.height

    def perimeter(self):
        return 2 * (self.width + self.height)

# Usage
rect = Rectangle(4, 5)
print(rect.area())       # 20
print(rect.perimeter())  # 18

Other Supporting Concepts

a. Composition

  • An alternative to inheritance where objects are composed of other objects.
  • Encourages a "has-a" relationship instead of an "is-a" relationship.
python
Copy code
class Engine:
    def start(self):
        return "Engine starting"

class Car:
    def __init__(self, engine):
        self.engine = engine

    def start(self):
        return self.engine.start()

# Usage
engine = Engine()
car = Car(engine)
print(car.start())  # Engine starting

b. Aggregation

  • A form of association where one object contains another, but they can exist independently.

Advantages of OOP

  1. Code Reusability: Through inheritance and modular design.
  1. Scalability: Easily extend classes or add new behaviors.
  1. Maintainability: Encapsulation ensures manageable and modular code.
  1. Flexibility: Polymorphism makes the code adaptable to new requirements.
  1. Abstraction: Reduces complexity by focusing on relevant details.

Limitations of OOP

  • Learning Curve: Steeper compared to procedural programming.
  • Performance Overhead: Abstraction and inheritance can make programs slower.
  • Overengineering Risk: Can lead to overly complex designs for simple tasks.

When to Use OOP?

  • When the problem domain naturally maps to objects.
  • For large, scalable, and maintainable applications.
  • When extensibility and reuse are priorities.

Summary

OOP principles—encapsulation, inheritance, polymorphism, and abstraction—help create structured, scalable, and maintainable software. By understanding these principles and applying them judiciously, developers can design systems that are robust, reusable, and easy to understand.
