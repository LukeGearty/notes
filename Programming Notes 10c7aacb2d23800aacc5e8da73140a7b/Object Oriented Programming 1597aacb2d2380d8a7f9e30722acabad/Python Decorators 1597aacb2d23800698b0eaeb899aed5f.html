Python Decorators

Notes on Python Decorators

Decorators

  • Definition: Decorators modify the behavior of functions or classes by wrapping them with additional functionality.
  • Usage: Functions are passed as arguments to the decorator function, wrapped in a "wrapper function," and the modified function is returned.

Syntax:

python
Copy code
@gfg_decorator
def hello_decorator():
    print("Gfg")

Equivalent to:

python
Copy code
def hello_decorator():
    print("Gfg")
hello_decorator = gfg_decorator(hello_decorator)

First-Class Functions

Python treats functions as first-class objects:

  1. Properties:
    • Functions are instances of the Object type.
    • Functions can be stored in variables, passed as arguments, and returned from other functions.
    • Functions can be stored in data structures like lists or dictionaries.
  1. Examples:
    • Storing Functions in Variables:
      python
Copy code
def shout(text):
    return text.upper()
yell = shout
print(yell("Hello"))  # Output: HELLO
    • Passing Functions as Arguments:
      python
Copy code
def greet(func):
    greeting = func("Hi!")
    print(greeting)
greet(str.upper)  # Output: HI!
    • Returning Functions:
      python
Copy code
def create_adder(x):
    def adder(y):
        return x + y
    return adder
add_15 = create_adder(15)
print(add_15(10))  # Output: 25

Behavior Modification with Decorators

  • Structure:
    • A decorator takes a callable function as an argument.
    • A wrapper function is defined to enhance the original function.
    • The wrapper function calls the original function and adds new functionality.

Example:

python
Copy code
def hello_decorator(func):
    def wrapper():
        print("Before execution")
        func()
        print("After execution")
    return wrapper

@hello_decorator
def greet():
    print("Hello!")
greet()
# Output:
# Before execution
# Hello!
# After execution

Practical Examples

  1. Timing Function Execution:
    python
Copy code
import time

def calculate_time(func):
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        end = time.time()
        print(f"Time taken: {end - start}")
        return result
    return wrapper

@calculate_time
def example_function():
    time.sleep(2)

example_function()
  1. Returning Values from Decorated Functions:
    python
Copy code
def hello_decorator(func):
    def wrapper(*args, **kwargs):
        print("Before execution")
        result = func(*args, **kwargs)
        print("After execution")
        return result
    return wrapper

@hello_decorator
def add(a, b):
    return a + b

print(add(3, 4))
# Output:
# Before execution
# After execution
# 7

Chaining Decorators

  • Multiple decorators can be applied to a function, and the order of application matters.
  • Example:
    python
Copy code
def decor1(func):
    def wrapper():
        return func() ** 2
    return wrapper

def decor2(func):
    def wrapper():
        return func() * 2
    return wrapper

@decor1
@decor2
def number():
    return 10

print(number())  # Output: 400

FAQs on Decorators

  1. When to Use Decorators?
    • Add logging, caching, authentication, or timing to functions.
    • Modify behavior without altering the original function's source code.
  1. Function vs. Decorator:
    • Function: Performs a task and may return a value.
    • Decorator: Enhances the behavior of a function without altering its source.
  1. Annotations vs. Decorators:
    • Annotations: Add metadata using : for arguments and > for return values.
      python
Copy code
def greet(name: str) -> str:
    return f"Hello, {name}"
    • Decorators: Modify function behavior.
  1. Wrapper vs. Decorator:
    • Wrapper: Inner function defined in a decorator to implement additional functionality.
    • Decorator: Outer function that takes another function, modifies it, and returns the wrapper.
