Active Directory (AD)

Overview:

  • Active Directory is a database used in environments with many Windows devices to centralize and manage all network components.
  • It contains information about:
    • Computers and devices on the network.
    • User accounts.
    • File shares and printers.
    • Security groups.
    • Other resources that make up the network.

Key Features:

  1. Centralized Authentication:
    • Authentication is managed centrally using the AD database.
    • Users log in to devices or authenticate to resources using their AD-defined username and password.
  1. Permission Management:
    • Access permissions can be assigned to individual users or groups.
    • Groups allow for easier management of permissions for multiple users.
  1. Account Management:
    • Functions like adding/removing accounts, modifying passwords, and managing access rights are performed via Active Directory.

Group Policy

Overview:

  • Group Policy is a feature layered over AD to enforce configurations and permissions on users and devices.
  • Managed using the Group Policy Management Editor.

Key Features:

  1. Configuration Settings:
    • Allows setting specific configurations for users and devices.
    • Examples:
      • Login scripts that execute when a user connects to the network.
      • Network configurations, such as Quality of Service (QoS).
      • Security parameters for users and devices.
  1. Security Policies:
    • Enables control over permissions and access rights for devices and users.
    • Supports enforcing company-wide or user-specific security standards.
  1. Centralized Control:
    • Combines AD and Group Policy to create a robust and comprehensive network control system.
    • Simplifies configuring security policies and settings for users and devices.

Linux Access Control

Default Access Control:

  • Discretionary Access Control (DAC):
    • The default access control model in Linux.
    • Users have the discretion to assign rights and permissions to resources they own.
    • Not ideal for highly secure environments due to lack of centralized control.

Mandatory Access Control (MAC):

  • Security-Enhanced Linux (SELinux):
    • SELinux modifies Linux to support mandatory access control instead of DAC.
    • All permissions and rights are centrally assigned by an administrator.
    • Promotes the principle of least privilege:
      • Users and processes are given only the permissions required to perform their tasks, minimizing risk.
    • Benefits:
      • Limits the scope of breaches, malware, or malicious activity on a Linux system.
      • Provides a more secure environment for sensitive operations.
    • Open Source:
      • SELinux is open source and can be installed on many Linux distributions.
