Wireless Network Security Concerns

  1. Over-the-Air Communication: Vulnerable to attackers who can intercept unencrypted data.
  1. Authentication: Preventing unauthorized access via usernames, passwords, or other methods.

Encryption and Integrity

  1. Default Encryption: Ensures intercepted data is unreadable.
  1. Message Integrity Check (MIC): Ensures data integrity during transmission.

WPA2 and Its Limitations

  1. Four-Way Handshake Vulnerability:
    • Attackers can capture the handshake and brute-force the pre-shared key (PSK) hash offline.
    • Modern tools (e.g., GPU processing, cloud cracking) accelerate brute-forcing.
  1. Shared PSK Limitation: Once known, any user can connect.

WPA3 Improvements

  1. Enhanced Security:
    • Replaces the four-way handshake with Simultaneous Authentication of Equals (SAE), using Diffie-Hellman key exchange.
    • Eliminates PSK hash transmission.
  1. GCMP:
    • Stronger encryption and built-in message integrity.
  1. Individual Session Keys: Unique keys for each user, enhancing privacy.

Authentication Methods

  1. WPA3-Personal (PSK):
    • Used in home networks with a shared password.
  1. WPA3-Enterprise (802.1X):
    • Used in corporate settings.
    • Requires unique usernames, passwords, and centralized authentication (e.g., RADIUS, LDAP).

AAA Framework (Authentication, Authorization, Accounting)

  1. Authentication: Verifies identity (e.g., username and password).
  1. Authorization: Defines accessible resources.
  1. Accounting: Logs metrics (e.g., session duration, data usage).

Centralized Authentication (802.1X and AAA)

  1. Devices Involved:
    • Supplicant: The user/device attempting to connect.
    • Authenticator: The network device (e.g., access point).
    • Authentication Server (AAA): Validates credentials.
  1. 802.1X Process:
    • Authenticator blocks access until credentials are provided.
    • Uses EAP (Extensible Authentication Protocol) for authentication.

Key Takeaways

  • WPA3 Advantages: Superior security, unique session keys, and protection against brute-force attacks.
  • Corporate Networks: Employ centralized authentication with AAA servers and 802.1X.
  • EAP: A flexible protocol enabling secure and customizable authentication.

RADIUS stands for Remote Authentication Dial-In User Service. Despite having "dial-in" in its name, RADIUS is a versatile protocol used for authentication, authorization, and accounting (AAA) in a variety of network scenarios, including local area networks (LANs), virtual private networks (VPNs), and wireless networks.

Key Points About RADIUS:

  1. Authentication:
    • Verifies the identity of users or devices attempting to connect to the network.
    • Typically involves a username and password being checked against the RADIUS server.
  1. Authorization:
    • Determines what resources the authenticated user can access on the network.
  1. Accounting:
    • Tracks user activity, such as connection time, data usage, and session duration.
  1. How It Works:
    • A RADIUS client (e.g., a router, switch, or VPN concentrator) forwards the authentication request from a user to the RADIUS server.
    • The RADIUS server checks the credentials against a user database and provides a response:
      • Accept: Grants access to the network.
      • Reject: Denies access.
      • Challenge: Requests additional information (e.g., multi-factor authentication).
  1. Common Uses:
    • Securing access to corporate wireless networks (commonly with WPA3-Enterprise).
    • Authenticating VPN users.
    • Centralized management of user credentials in an organization.
  1. Advantages:
    • Centralized Authentication: User credentials are managed on one server, making it easier to control access.
    • Scalability: Supports large networks with many devices.
    • Multi-Vendor Support: Widely supported by networking devices from various manufacturers.
  1. 802.1X Integration:
    • RADIUS is often used in conjunction with the 802.1X standard, which enforces authentication before granting network access.
  1. Common AAA Server:
    • RADIUS is one of the most widely used protocols for AAA servers, which may also use alternatives like LDAP or TACACS+.
