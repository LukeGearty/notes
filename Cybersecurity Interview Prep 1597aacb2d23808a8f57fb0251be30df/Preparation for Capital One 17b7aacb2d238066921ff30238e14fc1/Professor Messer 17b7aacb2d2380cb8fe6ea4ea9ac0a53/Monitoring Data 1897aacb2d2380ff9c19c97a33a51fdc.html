File Integrity Monitoring (FIM)

  • Purpose: Monitors critical files that should rarely change, alerts on unauthorized modifications.
  • Windows Utility:
    • System File Checker (SFC):
      • Scans and verifies integrity of critical OS files.
      • Replaces modified files with correct versions.
  • Linux Utility:
    • Tripwire:
      • Provides real-time monitoring of file changes.

Host-Based Intrusion Prevention Systems (HIPS)

  • Monitors and protects files on the OS.
  • Can block attacks against known vulnerabilities.
  • Differs from network-based IPS by operating at the host level.

Data Loss Prevention (DLP)

Key Functions:

  • Real-Time Traffic Monitoring:
    • Blocks sensitive data transmissions (e.g., SSNs, medical info).
    • Monitors:
      • Data in motion: Network traffic packets (e.g., integrated with NGFWs or standalone appliances).
      • Data in use: Data in active memory or being processed (endpoint DLP).
      • Data at rest: Files stored locally or on servers.

Endpoint DLP:

  • Controls USB drive usage:
    • DoD Example (2008): USB storage banned after worm virus outbreak via flash drives.
    • Restrictions revised in 2010 with stricter USB usage guidelines.

Cloud-Based DLP:

  • Monitors data in cloud-based applications.
  • Blocks sensitive info uploads and malware.

Email-Based DLP:

  • Outbound Email: Blocks sensitive data (e.g., SSNs, wire transfer fraud).
  • Inbound Email:
    • Detects suspicious keywords, spoofed emails.
    • Quarantines flagged emails.
  • Case Study: Boeing 2016 breach: Hidden fields in a spreadsheet exposed sensitive data of 36,000 employees.
