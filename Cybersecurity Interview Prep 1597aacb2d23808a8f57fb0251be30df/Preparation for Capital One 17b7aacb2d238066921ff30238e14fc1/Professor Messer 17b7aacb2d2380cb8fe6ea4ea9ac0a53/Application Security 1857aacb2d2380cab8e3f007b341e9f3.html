1. Application Vulnerabilities

  • Common vulnerabilities include buffer overflows, SQL injection, and other exploits.
  • Application development involves balancing speed and security.
  • Vulnerabilities can be found during:
    • Quality Assurance (QA): Functional and security checks.
    • Post-release: Discovered by researchers or attackers.

2. Input Validation

  • Ensures data input matches expected formats to prevent exploitation.
  • Examples:
    • Zip code fields require specific character formats.
    • Prompts users to correct invalid input.
  • Tools like fuzzers test input validation by injecting random data.

3. Cookies

  • Small data files stored in browsers for:
    • Tracking and personalization.
    • Maintaining login sessions.
  • Key facts:
    • Cookies are not executable but can reveal information.
    • Sensitive data should not be stored in cookies.
    • Secure cookies only transfer data over HTTPS.

4. Static Code Analysis (SAST)

  • Analyzes application code for vulnerabilities, e.g., buffer overflows and database injections.
  • Notable limitations:
    • Cryptographic issues often require deeper analysis.
    • False positives in static analyzer output require manual review.
  • Example: Replacing unsafe functions like gets() with safer alternatives like fgets().

5. Code Signing

  • Ensures code integrity and authenticity using digital signatures.
  • Process:
    • Developers sign code with a certificate from a trusted authority.
    • During installation, the OS verifies the signature.
    • Alerts users if the code has been altered.

6. Sandboxing

  • Restricts application access to only necessary data.
  • Types of sandboxing:
    • Development Sandbox: Isolated environment for app creation, preventing production network impact.
    • Runtime Sandbox: Protects system data from application misuse.
    • Example: Mobile browsers can access bookmarks but not personal photos.

7. Application Monitoring

  • Logs and monitors application usage and security events.
  • Detects:
    • SQL injection attempts.
    • Exploits of known vulnerabilities.
    • Anomalous activities, e.g., unusual file transfers or access spikes.
  • Logs help developers analyze and address potential threats.
