An on-path attack, formerly known as a man-in-the-middle (MITM) attack, occurs when an attacker intercepts and potentially alters communication between two parties without their knowledge. These attacks exploit the trust between communicating entities and often target vulnerabilities in protocols, systems, or networks.

Types of On-Path Attacks

  1. Passive Interception:
    • Description: The attacker eavesdrops on communication to steal sensitive information without altering the data.
    • Examples: Network sniffing, data harvesting.
    • Tools: Wireshark, Tcpdump.
  1. Active Interception:
    • Description: The attacker not only intercepts communication but also manipulates the data in transit.
    • Examples: DNS spoofing, HTTPS stripping.
    • Tools: Bettercap, Ettercap.

Common Techniques

  1. ARP Spoofing:
    • Description: Manipulating the Address Resolution Protocol (ARP) cache to associate the attacker’s MAC address with a legitimate IP address.
    • Impact: Redirects network traffic through the attacker’s machine.
    • Countermeasures: Enable Dynamic ARP Inspection (DAI) on switches, use static ARP entries.
  1. DNS Spoofing/Poisoning:
    • Description: Corrupting the DNS cache to redirect users to malicious websites.
    • Impact: Theft of credentials, phishing attacks.
    • Countermeasures: Use DNSSEC, implement secure recursive DNS resolvers.
  1. SSL Stripping:
    • Description: Downgrading HTTPS traffic to HTTP by intercepting and modifying initial connections.
    • Impact: Exposes sensitive data transmitted in plaintext.
    • Countermeasures: Use HSTS (HTTP Strict Transport Security), enforce HTTPS-only policies.
  1. Wi-Fi Eavesdropping:
    • Description: Intercepting unencrypted data on public or poorly secured Wi-Fi networks.
    • Impact: Credential theft, session hijacking.
    • Countermeasures: Use WPA3, avoid public Wi-Fi for sensitive tasks, and use VPNs.
  1. Email Hijacking:
    • Description: Intercepting and manipulating email communications, often through compromised servers or spoofing.
    • Impact: Fraud, phishing, or BEC (Business Email Compromise) scams.
    • Countermeasures: Implement email authentication (SPF, DKIM, DMARC), use end-to-end encryption.
  1. Session Hijacking:
    • Description: Stealing session tokens to impersonate a user in an ongoing session.
    • Impact: Unauthorized account access.
    • Countermeasures: Secure cookies with flags like HttpOnly, Secure, and SameSite.
  1. HTTPS Spoofing:
    • Description: Presenting a fraudulent SSL certificate to users to impersonate a trusted site.
    • Impact: Sensitive data theft.
    • Countermeasures: Educate users to verify SSL certificates, use certificate pinning.

Steps in On-Path Attacks

  1. Reconnaissance: Identifying the network, devices, or protocols to target.
  1. Interception: Gaining access to the communication channel (e.g., ARP spoofing, rogue AP).
  1. Exploitation:
    • Passive: Monitoring traffic for sensitive information.
    • Active: Injecting malicious payloads, altering data.
  1. Cleanup: Covering tracks to avoid detection.

Tools Used for On-Path Attacks

  1. Bettercap: Comprehensive MITM framework for ARP spoofing, DNS spoofing, and traffic manipulation.
  1. Ettercap: Network packet sniffing and injection tool.
  1. Wireshark: Network protocol analyzer for passive data interception.
  1. Responder: Captures NTLM hashes from broadcast traffic (e.g., LLMNR, NBNS).
  1. Evilginx2: Phishing tool leveraging reverse proxies to bypass MFA.

Countermeasures Against On-Path Attacks

  1. Encryption:
    • Use HTTPS with TLS 1.3.
    • Enforce end-to-end encryption (E2EE) in applications.
  1. Authentication and Integrity:
    • Implement strong authentication mechanisms (e.g., MFA).
    • Use digital signatures to verify message integrity.
  1. Secure Network Configuration:
    • Use VPNs for secure communication over untrusted networks.
    • Segment networks to limit attack surfaces.
  1. Secure Protocols:
    • Replace insecure protocols (e.g., FTP, Telnet) with secure ones (e.g., SFTP, SSH).
    • Use DNSSEC for DNS traffic.
  1. Monitoring and Detection:
    • Deploy Intrusion Detection Systems (IDS) and Intrusion Prevention Systems (IPS).
    • Regularly inspect SSL/TLS configurations and certificate validity.
  1. Education and Awareness:
    • Train users to recognize phishing attempts and verify secure connections.
    • Promote the use of security tools like password managers.

Emerging Trends

  1. AI-Driven MITM Attacks:
    • Using machine learning to analyze and manipulate intercepted data in real time.
  1. IoT On-Path Exploits:
    • Targeting IoT devices with weak security configurations.
  1. 5G-Related Threats:
    • Exploiting 5G protocol vulnerabilities to intercept communications.

ARP (Address Resolution Protocol) poisoning is a type of on-path attack where an attacker manipulates the ARP tables on a network to intercept or disrupt communication between devices. It exploits the lack of authentication in the ARP protocol, which resolves IP addresses to MAC addresses in local area networks (LANs).

Steps in ARP Poisoning

  1. Reconnaissance:
    • The attacker identifies target devices (e.g., a victim machine and a gateway/router) on the network.
    • Tools like arp-scan, nmap, or Netdiscover are used to map IP addresses to MAC addresses.
  1. Spoofing ARP Replies:
    • The attacker sends falsified ARP replies to the target devices, associating the attacker’s MAC address with the IP address of another device (e.g., the gateway/router).
    • For example:
      • Victim’s ARP table is updated to map the gateway’s IP to the attacker’s MAC.
      • Gateway’s ARP table is updated to map the victim’s IP to the attacker’s MAC.
  1. Interception:
    • As traffic flows between the victim and the gateway, it passes through the attacker’s device.
    • The attacker can:
      • Passively monitor (sniff) the traffic.
      • Modify the data before forwarding it to the intended recipient.
      • Drop packets to disrupt communication (Denial of Service).
  1. Persistence (Optional):
    • The attacker continually sends spoofed ARP replies to maintain control, as ARP caches can refresh or resolve conflicts.

Diagram of ARP Poisoning

  1. Normal Communication:
    • Victim ↔ Gateway (Direct traffic flow via legitimate MAC-IP mappings).
  1. After ARP Poisoning:
    • Victim ↔ Attacker ↔ Gateway (Traffic routed through the attacker).

Consequences of ARP Poisoning

  1. Data Interception:
    • Sensitive information such as login credentials, session cookies, or personal data can be captured.
  1. Data Manipulation:
    • The attacker can modify the contents of transmitted data (e.g., inject malware, perform phishing attacks).
  1. Denial of Service (DoS):
    • By not forwarding packets, the attacker can disrupt communication between devices.
  1. Session Hijacking:
    • The attacker can use intercepted session cookies to impersonate users.

Tools for ARP Poisoning

  1. Ettercap:
    • A suite for MITM attacks, including ARP poisoning.
  1. Bettercap:
    • A modern tool for ARP spoofing and network manipulation.
  1. Arpspoof:
    • A lightweight tool for sending forged ARP replies.

Countermeasures

  1. Dynamic ARP Inspection (DAI):
    • Ensures only valid ARP responses are processed by verifying IP-to-MAC bindings.
  1. Static ARP Entries:
    • Manually configure ARP tables for critical devices, preventing unauthorized changes.
  1. Encryption:
    • Use end-to-end encryption (e.g., HTTPS, VPNs) to secure data even if intercepted.
  1. Network Segmentation:
    • Isolate critical devices to reduce the attack surface.
  1. Monitoring and Alerts:
    • Deploy tools like arpwatch to detect unusual ARP activity.
  1. Implement IPv6:
    • IPv6 uses the Neighbor Discovery Protocol (NDP) instead of ARP, which includes built-in security features.
