Endpoint Security

The endpoint is the device used by the user to connect to the network

To stop the attackers, we need to monitor inbound and outbound attacks

Many different platforms susceptible to attacks

Protection is multi-faceted

Edge - the part of the network where the outside of the network meets the inside of the network - the internet link

Managed through firewall rules

Firewall rules rarely change

Posture Assessment - you need to constantly perform checks to make sure each system is up to date - done on all devices (desktops, laptops, mobile, etc)

A good time to conduct a posture assessment is before a device connects to the network

Persistent Agents - permanently installed onto a system that can conduct posture assessments - needs to be updated

Dissolvable Agents - no installation is required - runs during the posture assessment - terminates when no longer required

Agentless NAC - Active directory - checks are made during login and logoff

The security administrator decides what happens when a device does not pass the Posture Assessment

Quarantine network, notify admins

Endpoint Detection and Response (EDR)

A different method of threat protection

Detect a threat

Signatures aren’t the only detection tool - behavioral analysis, machine learning, process monitoring

Investigates the threat to figure out why - root cause analysis

Respond to the threat

Isolate the system

Quarantine the threat

Rollback to a previous config

API driven - no user or technician intervention required

Extended Detection and Response (XDR)

An evolution of EDR - improve missed detections, false positives, and long investigation times

Attacks involve more than just the endpoint

Endpoints and Monitoring

  • Endpoints: Devices like desktops, laptops, and mobile devices used by users.
    • Applications on endpoints can be exploited to access sensitive information.
    • Monitoring both inbound and outbound data is critical to detect malicious software.

Layered Security Approach

  • Different platforms (desktops, mobile phones, tablets) require defense in depth.
  • A layered security approach involves using multiple security solutions tailored to each platform.

Network Edge Security

  • The edge: Where the internal network connects to the internet.
  • Protected by firewalls using static rules to monitor traffic between the internal and external network.

Access Control

  • Limits device access to data using rules:
    • User identity: Username or group membership.
    • Location: Where the user or data resides.
    • Applications: Restricts access based on the application used.
  • Access control lists (ACLs) are dynamic and adjustable by administrators.

Posture Assessment

  • A process to ensure devices meet security standards.
    • Performed when a device connects or logs into the network.
    • Checks include:
      • Antivirus: Installed, running, and updated.
      • Applications: Updated to the latest versions.
      • Disk Encryption: For remote devices, full disk encryption is checked.
      • Trusted Device: Verified using company certificates.
  • Assessment methods:
    • Persistent agents: Software installed on devices, runs continuously.
    • Dissolvable agents: Temporary, runs during login or connection.
    • Agentless NAC (Network Access Control): Integrates with Active Directory for checks during login/logout.

Quarantine for Non-Compliant Devices

  • Non-compliant devices may be quarantined on a separate VLAN.
  • Users are guided to bring their devices into compliance before regaining full network access.

Antivirus and Modern Endpoint Security

  • Antivirus challenges:
    • Over 1 million virus variants are created daily.
    • Requires constant updates and scalability.
  • EDR (Endpoint Detection and Response):
    • Enhances antivirus by using:
      • Behavioral analysis.
      • Machine learning.
      • Root-cause analysis: Identifies how a virus entered and prevents re-infection.
    • Automates responses:
      • Isolates infected systems.
      • Quarantines malware.
      • Rolls back systems to a known-good state.

XDR (Extended Detection and Response)

  • Extends EDR capabilities by integrating data from multiple sources.
  • Key features:
    • Correlates data across devices and systems.
    • Identifies false positives more accurately.
    • Speeds up investigations through automation.
  • User Behavior Analytics (UBA):
    • Establishes a baseline of normal activity.
    • Detects anomalies using rules, pattern matching, and statistical analysis.

XDR Advantages

  • Monitors diverse data sources, including:
    • Network traffic.
    • User activity.
    • Device connections.
  • Correlates events to simplify threat identification and prevention in real-time.
