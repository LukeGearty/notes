1. Logging and Security Data Collection

  • Networks generate vast amounts of security-related log data.
  • Logs contain details such as:
    • Blocked and allowed traffic flows.
    • Exploit attempts, especially from Intrusion Prevention Systems (IPS).
    • Categories of URLs blocked on user workstations.
    • DNS sinkhole traffic, which may indicate malicious activity.

2. Firewall Logs

  • Firewalls monitor all inbound and outbound network traffic.
  • Logs include:
    • Source and destination IP addresses.
    • Port numbers and traffic flow status (allowed or blocked).
    • Next-Generation Firewalls (NGFWs) provide deeper insights:
      • Application identification.
      • URL filtering and threat detection.
      • Detection of anomalies in traffic.

3. Log Details in NGFWs

  • Each traffic flow log entry typically contains:
    • Timestamp.
    • Source & destination IP and MAC addresses.
    • Application in use.
    • Action taken by the firewall (allowed or blocked).

4. Application and OS Logs

  • Windows: Logs available in Windows Event Viewer.
  • Linux/macOS: Logs stored in /var/log/.
  • Logs are aggregated in Security Information and Event Management (SIEM) systems for centralized analysis.

5. Endpoint Device Logs

  • Laptops, desktops, mobile devices generate extensive logs:
    • Login and logout events.
    • System processes and security-related actions.
    • Device management logs (password changes, lockouts).
    • Directory service interactions.

6. SIEM and Log Correlation

  • SIEMs collect and correlate logs from multiple sources.
  • Helps track events across different devices and networks.
  • Security teams can monitor and respond to threats more effectively.

7. Operating System Security Logs

  • OS logs store critical security information:
    • Authentication events.
    • Changes to system files.
    • Unauthorized service modifications.
  • SIEM can alert security teams about anomalies in these logs.

8. Log Filtering for Efficiency

  • Due to the vast amount of log data, only relevant logs should be sent to the SIEM.
  • Filtering helps improve performance and decision-making.

9. Intrusion Detection and Prevention Logs

  • IDS/IPS systems provide logs for:
    • Known attack patterns.
    • Potential vulnerabilities.
    • DoS attacks (e.g., SYN flood).
  • Logs include:
    • Timestamp.
    • Attack classification and priority.
    • Source and destination details.

10. Network Infrastructure Logs

  • Devices like routers, switches, and VPN concentrators generate security logs.
  • Examples:
    • Routing table changes.
    • Authentication errors.
    • Network attack detection (e.g., TCP SYN floods).

11. Metadata in Digital Files

  • Files, emails, and images contain metadata that can reveal:
    • Device details.
    • GPS coordinates (for images).
    • Email headers (sender, path, authentication checks).
  • This data helps in forensic investigations and threat hunting.

12. Vulnerability Scan Logs

  • Logs from vulnerability scanners identify security weaknesses:
    • Missing firewalls or antivirus protection.
    • Misconfigurations (e.g., open shares, guest accounts enabled).
    • Unpatched operating systems and applications.

13. Automated SIEM Reports

  • SIEMs can generate automated security reports.
  • Challenges:
    • Reports need to be read and acted upon.
    • Processing large log datasets can be resource-intensive.
  • Reports should balance depth and efficiency.

14. SIEM Dashboards

  • Provide real-time insights into security events.
  • Useful for SOC teams to quickly assess threats.
  • Typically display:
    • System health.
    • Active firewall rules.
    • User and device activities.

15. Network Packet Analysis

  • Capturing packets can reveal security threats at the lowest level.
  • Tools like Wireshark help analyze:
    • Wired and wireless traffic.
    • HTTP requests, authentication attempts, and anomalies.
    • Detailed breakdown of network packet structures.
