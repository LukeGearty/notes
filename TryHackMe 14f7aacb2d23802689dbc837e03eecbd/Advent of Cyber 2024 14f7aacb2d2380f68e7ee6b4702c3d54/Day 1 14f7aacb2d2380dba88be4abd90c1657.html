Day 1

Notes on the Task

The Story

  • McSkidy investigates a suspicious website leading to the handle "Glitch."
  • The initial investigation appears simple, but the situation grows complex, revealing a deeper, tangled web of cyber threats.

Learning Objectives

  1. Investigate malicious link files.
  1. Understand OPSEC principles and mistakes.
  1. Learn to track and attribute digital identities in cyber investigations.

Steps in Investigation

  1. Connect to the Machine:
    • Start the virtual machine and AttackBox to access tools for analysis.
  1. Investigate the Website:
    • Website: A YouTube-to-MP3 converter shared during SOC-mas.
    • Risks Identified:
      • Malvertising.
      • Phishing scams.
      • Bundled malware.
  1. Download and Inspect Files:
    • Files extracted:
      • song.mp3 — A legitimate audio file.
      • somg.mp3 — A malicious .lnk (Windows shortcut) file.
    • Analysis Tools:
      • Use the file command to identify file types.
      • Use ExifTool to extract metadata and embedded commands.
  1. Key Findings in somg.mp3:
    • Embedded PowerShell command downloads and executes a malicious script:
      • Disables security restrictions.
      • Downloads and executes IS.ps1 to collect sensitive information (e.g., crypto wallets, browser credentials).

Further Investigation

  • Search for Code on GitHub:
    • Search unique strings from the script (e.g., "Created by the one and only M.M.").
    • Results reveal an OPSEC failure, linking the malicious actor to identifiable information.

Introduction to OPSEC

  • Definition:
    • Process of protecting sensitive information from adversaries.
  • Common OPSEC Mistakes:
    • Reusing usernames, emails, or handles across platforms.
    • Including identifiable metadata in code, documents, or images.
    • Posting publicly on forums or repositories with incriminating details.
    • Not using VPNs or proxies during malicious activities.

Real-World OPSEC Failures

  1. AlphaBay Admin Takedown:
    • Administrator linked to their identity by using personal email and Bitcoin accounts.
  1. APT1 Chinese Hacking Group:
    • Malware signed with real names, leading to identification.

Takeaways

  • Effective cyber investigations rely on exploiting OPSEC mistakes by malicious actors.
  • OPSEC awareness is critical to both attackers and defenders in cybersecurity.
