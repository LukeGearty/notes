Monitoring in an AWS Environment

  • AWS Cloud Service Provider (CSP): Care4Wares runs infrastructure on AWS (cloud environment).
  • Key AWS Tools for Investigation:
    • CloudWatch: Monitors and aggregates logs for applications.
    • CloudTrail: Tracks user/service actions within AWS.

CloudWatch Overview

  • Purpose: Centralized monitoring for metrics and logs.
  • Components:
    • Log Events: Single log entries (timestamped, with metadata).
    • Log Streams: Collection of log events from one source.
    • Log Groups: Logical groupings of log streams (e.g., logs from services running across hosts).
  • Requirement: CloudWatch agent must be installed to collect logs.

CloudTrail Overview

  • Purpose: Tracks actions performed by users, roles, or services.
  • Features:
    • Always On: Enabled by default for all users.
    • JSON-Formatted Logs: Events are recorded in JSON format.
    • Event History: Tracks 90 days of actions; custom trails allow extended retention.
    • Deliverable Logs: Can forward logs to CloudWatch for unified analysis.

Log Analysis Tools: JQ

  • What is JQ? Command-line JSON processor for filtering and transforming JSON data.
  • Basic Syntax:
    • View entire JSON array: jq '.[]' filename.json
    • Extract specific fields: jq '.[] | .field_name' filename.json
  • Application in Investigation: Extract meaningful information from CloudTrail logs.
