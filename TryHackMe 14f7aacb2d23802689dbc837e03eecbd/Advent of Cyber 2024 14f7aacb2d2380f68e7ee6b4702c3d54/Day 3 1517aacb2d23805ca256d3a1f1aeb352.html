Operation Blue: Investigating the Attack

Log Analysis Using ELK:

  • Purpose: Aggregates logs from multiple sources for easier analysis.
  • Interface:
    • Load logs in Kibana under the "Discover" tab.
    • Select the relevant log collection (e.g., "wareville-rails").
    • Adjust the time range for Oct 1, 2024, to review logs.

Kibana UI Components:

  1. Search Bar: Enter queries using KQL.
  1. Fields Pane: View parsed log fields (e.g., timestamps, IP addresses).
  1. Logs Pane: Inspect specific log entries.
  1. Filters: Narrow down results (e.g., filter by IP address or specific keywords).

KQL Syntax:

  • "value": Exact match search.
  • : Wildcard for partial matches.
  • OR/AND: Logical operators for combining queries.
  • :: Field-specific search (e.g., ip.address: "10.10.10.10").

Key Investigation Steps:

  1. Filter by suspicious IPs, e.g., 10.9.98.230.
  1. Examine activity timeline and logs.
  1. Search for indicators like file uploads (e.g., "shell.php" in log messages).
  1. Identify malicious activity patterns such as commands sent through a web shell.

Operation Red: The Attack Methodology

File Upload Vulnerabilities:

  • Websites often allow uploads for legitimate purposes but may mishandle:
    • File types.
    • File size.
    • File contents.

Attack Types:

  1. RCE: Uploading scripts to gain server control.
  1. XSS: Injecting malicious HTML or JavaScript.
  1. Web Shells: Scripts for remote server control.

Web Shells:

  • Purpose: Provide attackers with remote command execution and server control.
  • Capabilities:
    • Execute commands.
    • Manipulate files.
    • Launch attacks on other systems.

Weak Credentials:

  • Systems are often breached through default or weak login details:
    • Common usernames: admin, guest.
    • Common passwords: admin, 123456.

What is Remote Code Execution (RCE)?

  • Definition: Exploiting vulnerabilities to execute unauthorized code.
  • Impacts:
    • Server control.
    • Data exfiltration.
    • Network compromise.

Hands-On Practice
