Key Concepts

Detection Gaps

  1. Challenges:
    • Evolving threat techniques bypass detection.
    • Fine line between normal and malicious behaviors.
  1. Solutions:
    • Refine detection rules continually.
    • Cover gaps across the cyber kill chain phases.

Cyber Kill Chain

  • Standard attack phases:
    • Reconnaissance → Execution → Exploitation → Goal Execution.
  • Goal for Blue Teamers:
    • Detect threats early or at multiple phases to prevent successful execution.

Frameworks and Tools

MITRE ATT&CK Framework

  • Catalog of real-world tactics, techniques, and procedures (TTPs).
  • Provides theoretical insights but lacks practical testing guidance.

Atomic Red Team

  • A library of practical, executable test cases mapped to MITRE ATT&CK TTPs.
  • Features:
    • Manual and automated execution.
    • Identifies detection gaps and helps refine detection rules.

Practical Exercise

  1. Scenario: Investigate suspected misuse of the Atomic Red Team tool.
  1. Technique Tested: MITRE ATT&CK T1566.001 (Spearphishing with an Attachment).
  1. Steps:
    • Launch a PowerShell prompt as Administrator.
    • Use the Invoke-AtomicTest command with parameters:
      • AtomicTechnique: Specify the technique to test (e.g., T1566.001).
      • ShowDetails: Display detailed information about the test.
    • Simulate phishing attachment downloads and detect artifacts.

Command Example:

powershell
Copy code
Invoke-AtomicTest T1566.001 -ShowDetails

Key Takeaways

  • The Unified Cyber Kill Chain emphasizes phased detection.
  • The MITRE ATT&CK framework categorizes TTPs but requires complementary tools like Atomic Red Team for implementation.
  • Simulated attack scenarios using Atomic Red Team provide actionable insights for refining detection and response strategies.
