Questions on DNS

Basic DNS Concepts

  1. What is DNS?
    • Answer: DNS (Domain Name System) is a system that translates human-readable domain names (like www.example.com) into IP addresses (like 192.168.1.1) so devices can locate each other on the internet.
  1. What is the purpose of DNS?
    • Answer: DNS allows users to access websites and services using easy-to-remember domain names instead of having to remember numerical IP addresses.
  1. What is the role of the DNS resolver?
    • Answer: The DNS resolver is responsible for querying DNS servers to resolve domain names into IP addresses. It may use a cache to store previously resolved addresses to speed up future queries.

DNS Query Process

  1. What happens when you type a domain name into your web browser?
    • Answer: The browser sends a DNS query to resolve the domain name into an IP address. The DNS resolver checks its cache or queries DNS servers (root, TLD, and authoritative) until it finds the IP address.
  1. What are the steps involved in DNS resolution?
    • Answer:
      • The DNS resolver checks its cache.
      • If not found, it queries the root DNS server.
      • The root server directs the query to the appropriate TLD server.
      • The TLD server directs the query to the authoritative DNS server, which provides the IP address.

DNS Records

  1. What is an A record in DNS?
    • Answer: An A record (Address Record) maps a domain name to an IPv4 address (e.g., example.com → 192.168.1.1).
  1. What is a CNAME record used for?
    • Answer: A CNAME record (Canonical Name Record) is used to alias one domain name to another. For example, www.example.com can point to example.com.
  1. What is an MX record and what is its purpose?
    • Answer: An MX (Mail Exchange) record specifies the mail server responsible for receiving email for the domain. It helps route email to the correct server.
  1. What is the difference between an A record and an AAAA record?
    • Answer: An A record maps a domain to an IPv4 address, while an AAAA record maps a domain to an IPv6 address.
  1. What is a PTR record used for?
    • Answer: A PTR record (Pointer Record) is used for reverse DNS lookups, mapping an IP address to a domain name (e.g., 192.168.1.1 → example.com).

DNS Security

  1. What is DNSSEC and why is it important?
    • Answer: DNSSEC (DNS Security Extensions) adds digital signatures to DNS records to verify their authenticity and prevent tampering, ensuring data integrity and protection against DNS spoofing.
  1. What is DNS spoofing?
    • Answer: DNS spoofing (or cache poisoning) is an attack where malicious DNS records are inserted into a cache, causing users to be redirected to fraudulent websites.

DNS Configuration

  1. What is the purpose of a nameserver (NS) record?
    • Answer: An NS (Name Server) record specifies which DNS servers are authoritative for a particular domain and can provide information about that domain.
  1. What is the default port used for DNS?
    • Answer: DNS typically uses port 53 for both UDP and TCP.

DNS Tools and Troubleshooting

  1. What command-line tool can you use to query DNS records?
    • Answer: Tools like nslookup and dig are used to query DNS records and troubleshoot DNS issues.
  1. What might cause slow DNS resolution?
    • Answer: Slow DNS resolution can be caused by an overloaded DNS server, network issues, or DNS misconfiguration (e.g., incorrect TTL values or outdated records).
  1. What does TTL (Time to Live) mean in DNS?
    • Answer: TTL (Time to Live) specifies how long a DNS record is cached by a DNS resolver before it expires and a new query must be made to retrieve the updated record.

Advanced DNS Concepts

  1. What is the difference between a recursive DNS server and an authoritative DNS server?
    • Answer:
      • A recursive DNS server queries other DNS servers to resolve domain names and returns the result to the client.
      • An authoritative DNS server contains the original, authoritative records for a domain and provides the IP address when queried.
  1. What is the root DNS server's role in the DNS hierarchy?
    • Answer: The root DNS servers are the highest level in the DNS hierarchy and direct queries to the appropriate TLD (Top-Level Domain) DNS servers, which then point to authoritative DNS servers.
  1. What is DNS propagation and how long does it take?
    • Answer: DNS propagation is the process of updating DNS records across the internet. It can take anywhere from a few minutes to 48 hours for changes to fully propagate.
