Network Edge

clients are our personal devices

servers are also at the edge - maybe at the opposite sides of the edge

Network Core

A series/mesh of interconnected routers - nothing but routers

Packet-Switching: hosts break application-layer messages into packets (digitized into bits)

The network core forwards packets from one router to the next, across links on path from source to destination

Each packet is transmitted at full link capacity

Packets cannot travel along the same link simultaneously

They can if there is a different frequency, but not along the same frequency

Routers operate at layer 3, whether they are in the edge or core

Store-and-Forward

The entire packet (all of its bits) must arrive at the router before it can be transmitted on next link

If we have a packet of L bits leaving from one computer to another, all L bits must arrive att he router before being transmitted

Transmission Delay: takes L/R seconds to transmit (push-out) L bit packet into link at R bps

Packet-Switching: queuing delay, loss

Packet queuing and loss: if arrival rate (in bps) to link exceeds transmission rate (bps) of link for a period of time

Packets will queue, waiting to be transmitted on output link

Packets can be dropped (lost) if memory (buffer) in router fills up

Two Key network-core functions

Forwarding:

Each input interface has a forwarding table that has a header value and the output link the packet needs to leave from

Local action: move arriving packets from router’s input link to appropriate router output link

Routing:

Global action: determine source-destination paths taken by packets

Routing algorithm

Alternative to packet switching: circuit switching

end-end resources are allocated to, reserved for “call” between source and destination

Dedicated resources: no sharing

Circuit-like guaranteed performance

If there are no resources, then they need to wait

FDM and TDM

Frequency Divison Multiplexing:

optical, electromagnetic frequencies are divided into narrow frequency bands

Each call allocated its own band, can transmit at max rate of that narrow band

If the transmission frequency is 8 Mbps, and there are 4 users, each band will transmit 2 Mbps

Since there is a continuous need for transmission, things like VoIP, police radio, streaming can be good for FDM

Time Division Multiplexing (TDM)

Time divided into slots

Each call allocated periodic slot(s), can transmit at maximum rate of (wider) frequency band, but only during its time slot(s)

Email can used TDM - something that doesn’t require ongoing communication - something that is asynchronous

Packet-Switching versus Circuit Switching

Packet switching allows more users to use network

1 Gb/s link

active 10% of the time

Circuit Switching: 10 users

Packet Switching: with 35 users, probability > 10 active at same time is less. than .0004 *

Packet switching is great for bursty daya - sometimes has data to send, other times not

Resource sharing

Simpler, no call setup

With packet switching, excessive congestion is possible: packet delay and loss due to buffer overflow

Protocols needed for reliable data transfer, congestion control

Internet Structure: a “network of networks”

Hosts connect to internet via access Internet Service Providers (ISPs)

Residential, enterprise ISPs

Access ISPs in turn must be interconnected so that any two hosts can send packets to each other

Resulting network of networks is very complex - evolution was driven by economics and national policies

How do we connect millions of access ISPs together?

Connecting each access ISP to each other doesn’t scale - Big O (N^2)

Multiple ISPs with multiple competitors

Each country can have multiple ISPs

We establish IXP (Internet Exchange Points) and Peering Links ( direct link between two networks that allows them to exchange data without going through other networks) to connect them together

We as consumers need to pay for access to the internet, and the ISPs then allocate for us resources to do that.

Content Provider Networks -  distributed network of servers strategically located around the world that store copies of website content- Google, Microsoft, Akamai, etc that can charge consumers directly to run their own network, to bring services, content close to end users

ISPs still need to carry all our traffic, and content providers don’t need to pay anything to the ISPs.

Is this fair? No

It puts all the cost on the consumers
