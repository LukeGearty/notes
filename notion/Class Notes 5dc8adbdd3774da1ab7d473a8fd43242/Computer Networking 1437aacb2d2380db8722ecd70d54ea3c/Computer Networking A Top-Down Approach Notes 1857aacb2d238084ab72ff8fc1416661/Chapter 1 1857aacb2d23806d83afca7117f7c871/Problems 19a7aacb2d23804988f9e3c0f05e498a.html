P1.  Design and describe an application-level protocol to be used between an automatic teller machine and a bank’s centralized computer. Your protocol should allow a user’s card and password to be verified, the account balance (which is maintained at the centralized computer) to be queried, and an account withdrawal to be made (that is, money disbursed to the user). Your protocol entities should be able to handle the all-too-common case in which
there is not enough money in the account to cover the withdrawal. Specify
your protocol by listing the messages exchanged and the action taken by the
automatic teller machine or the bank’s centralized computer on transmission
and receipt of messages. Sketch the operation of your protocol for the case of
a simple withdrawal with no errors, using a diagram similar to that in Figure 1.2.
Explicitly state the assumptions made by your protocol about the underlying
end-to-end transport service.

Message from ATM machine to Server:

HELO <userid> - lets server know there is a card in ATM. UserID is transmitted to server

PASSWD <passwd> - user transmits PIN, which is sent to server

BALANCE - user requests balance amount

WITHDRAWL <amount> - user requests to withdraw money

CLOSE - user is done

Message from Server to ATM Machine

PASSWD - asking user for PIN number

OK - requested operation is valid

ERROR - requested operation is invalid

BALANCE <balance> - requested balance

AMOUNT <amt> - requested withdrawal amount

CLOSE - user is done, closing the transaction

Correct Operation:

client → HELO <userid> → server

server → PASSWD → client

client → PASSWD <passwd> → server

server → OK → client

client → BALANCE → server

server → BALANCE <balance> → client


P2.  
Equation 1.1 gives a formula for the end-to-end delay of sending one packet of length L over N links of transmission rate R. Generalize this formula for sending P such packets back-to-back over the N links.


Total delay for packet of length L to be transmitted at rate R over N links is equal to N x L/R

The first packet arrives at N(L/R) time. The second packet moves right after the first one, so it arrives at the destination (L/R) time after the first.

The third packet arrives at L/R time after the second.

This continues for all P packets.

The first packet reaches there at N(L/R).

The second packet reaches there at N(L/R) + (L/R).

The third packet reaches there at N(L/R) + 2(L/R).

This keeps going until the last (P-th) packet arrives.

For the P-th packet, the total time is equal to N(L/R) + (P-1)(L/R)
