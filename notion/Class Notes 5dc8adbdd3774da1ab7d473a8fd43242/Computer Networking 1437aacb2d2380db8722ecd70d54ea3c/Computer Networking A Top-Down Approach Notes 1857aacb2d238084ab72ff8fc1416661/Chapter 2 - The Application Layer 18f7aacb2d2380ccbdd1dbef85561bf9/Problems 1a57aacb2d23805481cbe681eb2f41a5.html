P6. Obtain the HTTP/1.1 specification (RFC 2616). Answer the following questions:


a. Explain the mechanism used for signaling between the client and server to indicate that a persistent connection is being closed. Can the client, the server, or both signal the close of a connection?

Persistent connections are the default behavior of HTTP/1.1. Servers assume that the client intends to maintain a persistent connection unless a Connection header including the connection-token close was sent in the request. If the server chooses to close the connection immediately after sending the response, it should send a Connection header including the connection-token close.

b. What encryption services are provided by HTTP?

None

c. Can a client open three or more simultaneous connections with a given
server?

No: Section 8.1.4 states: “single-user client should not maintain more than 2 connections with any server or proxy.”

d. Either a server or a client may close a transport connection between them
if either one detects the connection has been idle for some time. Is it
possible that one side starts closing a connection while the other side is
transmitting data via this connection? Explain.

Yes. 8.1.4 states: “

A client, server, or proxy MAY close the transport connection at any
   time. For example, a client might have started to send a new request
   at the same time that the server has decided to close the "idle"
   connection. From the server's point of view, the connection is being
   closed while it was idle, but from the client's point of view, a
   request is in progress.

P7.  Suppose within your Web browser you click on a link to obtain a Web page.
The IP address for the associated URL is not cached in your local host, so
a DNS lookup is necessary to obtain the IP address. Suppose that
n DNS
servers are visited before your host receives the IP address from DNS; the
successive visits incur an RTT of RTT1, . . . , RTT
n. Further suppose that the
Web page associated with the link contains exactly one object, consisting of
a small amount of HTML text. Let RTT0 denote the RTT between the local
host and the server containing the object. Assuming zero transmission time
of the object, how much time elapses from when the client clicks on the link
until the client receives the object?

Since the IP address is not cached, the local host must query multiple DNS servers sequentially. The query travels through n DNS servers, each adding a round trip time. The total time spent in the DNS lookup process is:

RTT1 + RTT2 + … + RTTn

Once the IP address is resolved, the client initiates a TCP connection to the web server. A TCP connection typically requires a 3-way handshake. That is RTT0.

Then the client sends an HTTP request and gets a response. That is RTT0.

So the total is 2RTT0 + (RTT1 + RTT2 + … RTTn)
