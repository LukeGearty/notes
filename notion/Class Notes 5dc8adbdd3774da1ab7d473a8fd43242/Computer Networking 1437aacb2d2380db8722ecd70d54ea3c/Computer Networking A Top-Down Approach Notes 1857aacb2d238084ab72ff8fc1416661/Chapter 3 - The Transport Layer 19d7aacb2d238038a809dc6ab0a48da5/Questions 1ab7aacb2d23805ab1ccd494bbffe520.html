Sections 3.1-3.3

R3.  Consider a TCP connection between Host A and Host B. Suppose that the TCP segments traveling from Host A to Host B have source port number x and destination port number y. What are the source and destination port numbers for the segments traveling from Host B to Host A?
The source port will be y, and the destination port will be x

R4.  Describe why an application developer might choose to run an application over UDP rather than TCP.

If that application developer wants to prioritize speed over delivery, and if packet loss isn’t a huge issue for that application, they might choose UDP over TCP.

R5.  Why is it that voice and video traffic is often sent over TCP rather than UDP
in today’s Internet? (
Hint: The answer we are looking for has nothing to do
with TCP’s congestion-control mechanism.)
Many firewalls are configured to block UDP traffic - TCP lets it get through


R7.  Suppose a process in Host C has a UDP socket with port number 6789. Suppose both Host A and Host B each send a UDP segment to Host C with destination port number 6789. Will both of these segments be directed to the same socket at Host C? If so, how will the process at Host C know that these two segments originated from two different hosts?

Yes, those two UDP segments from different hosts will go to the same socket. The operating system will provide the process with the IP addresses to determine the origin of the individual segments.

Section 3.4

R10. In our rdt protocols, why did we need to introduce timers?

Timers were introduced to provide a time for when packets needed to be retransmitted because of packet loss.

R12.  Visit the Go-Back-N interactive animation at the companion Web site.

  1. Have the source send five packets, and then pause the animation before
    any of the five packets reach the destination. Then kill the first packet and
    resume the animation. Describe what happens.
    After not getting the first packet, the receiver does not send any ACKs to the sender. After some time, because the sender did not receive ACKs for any packets, the 5 packets are retransmitted.
  1. Repeat the experiment, but now let the first packet reach the destination
    and kill the first acknowledgment. Describe again what happens.
    Because the other ACKs, up to packet 4, were received, the window still moves.
  1. Finally, try sending six packets. What happens?
    The window is set to 5, so that cannot happen.

R13.  Repeat R12, but now with the Selective Repeat interactive animation. How
are Selective Repeat and Go-Back-N different?

  1. The other packets arrive and receive ACKs, but those packets are buffered until the first packet arrives. The window does not move until the ACK for the first packet arrives.
  1. After some time, the sender will resend the packet to get the ACK for the first packet
  1. The window is still set to 5
