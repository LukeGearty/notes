Introduction to the Link Layer

Any device that runs a link-layer protocol is a node

Nodes can include hosts, routers, switches, WiFi Access Points.

The communication channels that connect adjacent nodes along the communication path are links.

In order for a datagram to be transferred from host to destination ,it needs to be sent over each individual link in the end-end path.

Over a given link, a transmitting node encapsulates the datagram in a link-layer frame and transmits the frame into the link.

The Services Provided by the Link Layer

  • Framing: Most link-layer protocols encapsulate each network-layer datagram within a link-layer frame before transmission voer the link. The frame consists of:
    • Data: The network-layer datagram is inserted.
    • Number of header fields.
    • Structure is specified by the link-layer protocol
  • Link Access: A medium access control protocol specifies the rules by which a frame is transmitted onto the link. For point-to-point links that have a single sender at one end of the link and single receiver at the other end, the MAC protocol is simple - the sender can send a frame whenever the link is empty.
    • When multiple nodes sharw a single broadcast link, that is called the multiple access problem.
  • Reliable Delivery: A link-layer protocol can provide reliable delivery service. TCP also provides a reliable delivery service, and link-layer can provide it with acknowledgments and retransmissions.
    • Often used on links prone to high error, like wireless.
    • Can be considered unnecessary for low bit-error links - fiber, coax, twisted-pair.
    • Many link-layer protocols don’t provide reliable delivery.
  • Error detection and correction: Link-layer protocols can provide a mechanism to detect bit errors, because there is no need to forward datagrams with bit errors.

Where is the Link layer Implemented?

Ethernet capabilities are either integrated into the motherboard chipset or implemented via a low-cost dedicated Ethernet chip - network adapter, also known as Network Interface Controller (NIC).

Much of it is implemented in hardware.

On the sending side:

The controller takes a datagram that has been created and stored in host memory by the higher layers of the protocol stack, encapsulates the datagram in a link-layer frame, and then transmits the frame into the communication link, following the link-access protocol.

On the Receiving Side:

A controller receives the entire frame and extracts the network layer datagram.

If the link layer performs error detection:

It is the sending controller that sets error-detection bits in the frame header and it is the receiving controller that performs error detection.

Part of the link layer is implemented in software that runs on the host’s CPU. These software components perform higher-level link-layer functionality such as assembling link-layer addressing info and activating the controller hardware. On the receiving side, the software responds to controller interrupts, handling error conditions and passing a datagram up to the network layer.

Error-Detection and -Correction Techniques

Error detection and correction services are also often offered at the transport layer.

At the sending node, data D is sent. This is to be protected agaisnt bit errors with error-detection and -correction bits (EDC). The data is not only the datagram from the network layer, but also link-level addressing info, sequence numbers, and other fields. D and EDC are sent on the receiving node in a frame.

At the receiving node, D’ and EDC’ are received. The receiver must determine whether or not D’ is the same as the original D. Error-detection and -correction can sometimes (not always) detect bit errors. It may not always be aware of bit errors. So any error detection schemes must keep the probability of undetected errors going through small.

Parity Checks

A single parity bit is added to data D. In an even parity scheme, the sender includes one additional bit and chooses its value such that the total number of 1s in the D + 1 bits is even. For odd parity schemes, the parity bit value is chosen such that there is an odd number of 1s.

The receiver counts the number of 1s in the received D + 1 bits. If an odd number of 1-valeud bits are found with even, the receiver knows that at least one bit error has occured. More precisely, it knows that some odd number of bit errors have occured.

If there is an even number of bit errors, this would result in an undetected error.

Errors are often clustered together in “bursts”. In a burst error condition, the probability of undetected errors in a frame protected by single-bit parity can aprpoach 50 percent.

Two-dimensional parity imrpoves on single parity by enabling both detection and correction of single-bit errors, and detection of two-bit errors.

Forward Error Correction (FEC) is the ability to both detect and correct errors. These techniques are commonly used in audio storage and playback devices like audio CDs.

Checksumming Methods

In checksumming, the D bits of data are treated as a sequence of k-bit integers.

Cyclic Redundancy Check (CRC)

Also known as polynomial codes, since it is possible to view the bit string to be sent as a polynomial whose coefficients are the 0 and 1 values in the bit string, with operations on the bit string interpreted as polynomial arithmetic.

the d-bit piece of data, D, that the sending node wants to send to the receiving node.

The sender and receiver first agree on an r + 1 bit pattern, known as a generator, G.

The most significant (leftmost) bit of G must be a 1. For a given piece of data, the sender will choose r additional bits, R, and append them to D such that the resulting d + r bit pattern is exactly divisible by G using modulo-2 arithmetic.

Modulo 2 arithmetic means that addition and subtraction are identical, and both are identical to XOR of the operands.

1011 XOR 0101 = 1110
1001 XOR 1101 = 0100

1011 - 0101 = 1110
1001 - 1101 = 0100

Multiple Access Links and Protocols

Two types of network links:

point-to-point: a single sender at one end of the link and a single receiver at the other end

Broadcast Link: multiple sending and receiving nodes all connected to the same, single shared broadcast channel.

Multiple Access Problem: The problem of how to coordinate the access of multiple sending and receiving nodes to a shared broadcast channel

Multiple Access Protocols: protocols by which nodes regulate their transmission into the shared broadcast channel

To decide who gets to talk and when, and who receives it

Human Example: “Raise your hand if you have a question”, “Give everyone a chance to speak”, etc.

Multiple Access Protocols can be classified into three categories:

Channel Partitioning Protocols

Random Access Protocols

Taking-Turns Protocols

Ideally, a multiple access protocol for a broadcast channel of rate R bits per second should have these characteristics:

  1. When only one node has data to send, that node has a throughput of R bps - the entire throughput to itself
  1. When M nodes have data to send, each of these nodes has a throughput of R/M bps. This need not necessarily imply that each of the M nodes always has an instantaneous rate of R/M, but rather that each node should have an average transmission rate of R/M over some suitably defined interval of time.
  1. The Protocol is decentralized; that is, there is no master node that represents a single point of failure for the network
  1. The protocol is simple so that it is inexpensive to implement

Channel Partitioning Protocols

Time Division Multiplexing and Frequency Division Multiplexing can partition a broadcast channel’s bandwidth among all nodes sharing that channel.

Suppose a channel supports N Nodes and the transmission rate is R bps.

TDM divides time into time frames and further divides each time frame into N time slots. Each time slot is assigned to one of the N nodes. When a node has a packet to send, it transmits the packet’s bits during its assigned time slot in the revolving TDM frame.

TDM eliminates collisions and is perfectly fair. But it has two major drawbacks:

  1. A node is limited to an average of R/N bps even when it is the only node with a packet to send.
  1. A node must always wait for its turn in the transmission sequence even when it is the only node with a frame to send.

FDM divides the R bps channel into different frequencies and assigns each frequency to one of the N nodes. FDM thus creates N smaller channels of R/N bps out of the single, larger R bps channel. FDM shares advantages and drawbacks with TDM.

Code Division Multiple Access (CDMA) - assigns a different code to each node. Each node uses its unique code to each node. The node then uses its unique code to encode the data bits it sends.

CDMA codes must be chosen carefully, because CDMA networks can have the property that each node can transmit simultaneously and have their respective receivers correctly receive the bits in spite of that.

Random Access Protocols

In a Random Access Protocol, the transmitting node always transmits at the full rate, R bps, of the channel. When there is a collision, each node involved in the collision repeatedly retransmits its packet until it gets through without a collision. When a node experiences a collision, the node waits a random delay before retransmitting the frame.

Like trying to get through a door at the same time as someone else. You both will awkwardly wait for the other person to go through until someone eventually does.

Slotted ALOHA

Assume all of the following:

  • All frames consist of exactly L bits.
  • Time is divided into slots of size L/R seconds (that is, a slot equals the time to transmit one frame).
  • Nodes start to transmit frames only at the beginnings of slots.
  • The nodes are synchronized so that each node knows when the slots begin.
  • If two or more frames collide in a slot, then all the nodes detect the collision event before the slot ends.

Let p be a probability, a number between 0 and 1.

When the node has a fresh frame to send, it waits until the beginning of the next slot and transmits the entire frame in the slot.

If there isn’t a collision, the node has successfully transmitted its frame and thus need not consider retransmitting the frame.

If there is a collision, the node detects the collision before the end of the slot. The node retransmits its frame in each subsequent slot with probabilty p until the frame is transmitted without a collision.

The event of retransmission occurs with probability p. The event of skip the slot and try to retransmit again corresponds to probability (1 - p)

Advantages of Slotted ALOHA:

  1. Allows a node to transmit continuously at the full rate when it is the only active node
  1. Highly decentralized

When there are multiple active nodes, a certain fraction of the slots will have collisions and will therefore be “wasted”. Another fraction of the slots will be empty because all active nodes refrain from transmitting as a result of the probabilistic transmission policy.

ALOHA

In pure ALOHA, when a frame first arrives, the node immediately transmits the frame in its entirety into the broadcast channel.

Carrier Sense Multiple Access (CSMA)

CSMA embodies two important “human” protocols:

  1. Listen before speaking
  1. If someone is taking at the same time, stop talking

Taking Turns Protocols

ALOHA and CSMA protocols have the property of:

  1. When only one node has data to send, that node has a throughput of R bps - the entire throughput to itself

But they do not have the property of:

2. when M nodes are active, then each active node has a throughput of nearly R/M bps

This motivated researchers to create another class of protocols, the taking-turns protocols.

Polling Protocol: requires one of the nodes to be designated as master node, and then it polls each of the nodes in a round-robin way.. The master node sends a message to node 1, saying it can transmit up to some maximum number of frames. After node 1 does that, the master node then messages node 2 to tell it can transmit up to a maximum number of frames. This continues in a cyclical manner.

Eliminates collisions and empty slots, gets a much higher efficiency, but has a few drawbacks:

  1. Polling Delay is introduced
  1. If the master node fails, the entire channel can fail.

Bluetooth is an example of a polling protocol.

Token passing Protocol: a small special-purpose frame known as a token is exchanged among the nodes in some fixed order. Node 1 might always send to node 2, node 2 to node 3, and node N back to Node 1. When the node receives the token, it only holds on if it has some frames to transmit, otherwise it forwards the token.

Switched Local Area Networks

A switched local area network connects three departments, two servers, and a router with four switches. The switches don’t recognize network-layer addresses, so they forward link-layer frames using link-layer addresses.

Link-Layer Addressing and ARP

All hosts and routers have link-layer addresses and network-layer addresses.

MAC Addresses

Hosts and routers don’t really have link-layer addresses, it is instead the adapters that have them (network interfaces).

It is the job of the link-layer switch to carry datagrams between hosts and routers.

A link-layer address is called: LAN address, physical address, MAC address. MAC address is the most popular term.

The MAC Address is 6 bytes long, giving 2^48 possible MAC addresses. These 6 bytes are expressed in hexadecimal notation. MAC addresses were originally designed to be permanent, but it is possible to change an adapter’s MAC address.

IEEE manages the MAC address space. When a company wants to manufacture adapters, it purchases a chunk of the address space.

An adapter’s MAC address doesn’t change no matter where it goes. It doesn’t have a hierarchical structure, unlike IP addresses’s network part and host part.

The broadcast MAC address is FF-FF-FF-FF-FF-FF.

Address Resolution Protocol (ARP)

The protocol that translates between IP address and MAC addresses (network level and link-layer level)

To discover the MAC addres for a destination in the same LAN, an ARP module in the sending host takes any IP address on the same LAN as input and returns the corresponding MAC address.

Each host and router has an ARP table in its memory, which contains mapping of IP addresses to MAC addresses.

Query ARP message is sent within a broadcast frame, whereas the response is sent within a standard frame.

ARP is plug and play.

Sending a Datagram off the Subnet

When a Host on a subnet wants to send a network-layer datagram to a host off the subnet (across a router).

In order for a datagram to be sent to a host in another subnet, the datagram needs to be sent to the router interface, so the appropriate MAC address is that router interface.

Ethernet

The most widely used wired LAN solution.

A hub is a physical layer device that acts on individual bits rather than frames. In 1990s, an ethernet installation was connected directly to a hub with twisted-pair copper wire.

The hub was replaced with a switch, which is “collision-less” and a bona-fide store-and-froward packet switch that operates up through layer 2.

Ethernet Frame Structure

There are six fields of the ethernet frame:

  • Data Field (46 to 1500 bytes) - carries the IP datagram. Maximum Transmission Unit of Ethernet is 1500 bytes.
  • Destination Address (6 bytes): contains the MAC address of the destination adapter.
  • Source Address (6 bytes): Source MAC address that transmitted the frame.
  • Type Field (2 bytes) : permits Ethernet to multiplex network-layer protocols.
  • Cyclic Redundancy Check (CRC) (4 bytes)
  • Preamble (8 bytes): The ethernet frame begins with the preamble.

Ethernet Technologies

Ethernet comes in many different flavors.

Link-Layer Switches

The role of the switch is to receive incoming link-layer frames and forward them onto outgoing links.

The switch is transparent to the hosts and routers in the subnet; a host/router addresses a frame to another host/router and happily sends the frame into the LAN, unaware that a switch will be receiving the frame and forwardign it.

Forwarding and Filtering

Filtering: The switch function that determines whether a frame should be forwarded to some interface or should just be dropped.

Forwarding: the switch function that determines the interfaces to which a frame should be directed and then moves the frame to those interfaces.

Switch Table: switch filtering and forwarding is done with a table that contains entries for some, but not necesarily all, of the hosts and routers on a LAN.

Self-Learning

A switch table is built automatically, dynamically, and autonomously without any intervention from a network administrator or from a configuration protocol.

Properties of Link-Layer Switching

  • Elimination of Collisions: there is no wasted bandwidth due to collisions. The switches buffer frames and never transmit more than one frame on a segment at any one time.
  • Heterogeneous Links: different links in the LAN can operate at different speeds and can run over different media.
  • Management: A switch eases network management.

Switches Versus Routers

Switches forwards packets using MAC address.

Switches are Layer-2.

Virtual Local Area Networks (VLANs)

Three drawbacks to the configuration of a LAN:

  • Lack of traffic isolation: broadcast traffic (frames carrying ARP or DHCP messages for example or frames whose destination has not yet been learned by a self-learning switch) must still traverse the entire institutional network.
  • Inefficient use of Switches
  • Managing Users: Employees belonging to two groups in a LAN makes the problem harder

Virtual Local Area Networks: A switch that supports VLANs allows multiple virtual LANs to be defined over a single physical LAN infrastructure. Hosts within a VLAN communicate with each other as if they were connected to the switch.
