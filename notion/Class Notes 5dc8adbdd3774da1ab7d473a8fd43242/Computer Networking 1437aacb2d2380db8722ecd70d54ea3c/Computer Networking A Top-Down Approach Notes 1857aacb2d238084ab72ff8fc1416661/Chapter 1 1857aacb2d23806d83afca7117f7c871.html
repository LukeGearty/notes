1.1 What is the Internet?

Nuts and Bolt Description - A computer network that interconnects billions of computers, workstations, servers, mobile devices, gaming consoles, IoT devices, and others together. These devices are called hosts/end systems.

A network can be a mobile network, connecting together cars, traffic lights, smartphones, and laptops.

Or it can be a home network connecting desktops, a laptop, and IoT devices.

It can be an enterprise network connecting together hundreds of desktops, laptops, tablets, and servers.

It can be a content provider network that connects Datacenter networks.

All are connected to a local or regional ISP, which is connected to a national or global ISP.

How do these systems connect together?

Through communication links - physical cables and radio signals that can transmit data at different rates measured in bits/second. The bits of information are called packets.

Different links can transmit data at different rates, with the transmission rate of a link measured in bits/second.

When data is being sent to another end system, the sender segments the data and adds header bytes to each system to make a packet.

Packet Switches - take a packet arriving on one of its incoming communication links and forwars the packets on one of its outgoing communication links. There are routers and link-layer switches.

Link layer switches are used in access networks, while routers are used in the network core.

The sequence of links and packet switches traversed by a packet from the sending end system to the receiving end system is known as a route or path through the network.

End systems access the Internet through ISPs. Residential ISPs (local cable or phone companies), corporate ISPs, university ISPs, ISPs that provide WiFi access in places like airports or hotels or shops, and cellular data ISPs.

Pieces of the Intern run protocols that control the sending and receiving of information within the Internet.

Services Description - a view of the internet as an infrastructure that provides services to applications.

Distributed Applications involve multipel end systems that exchange data with each other.

Applications are typically written on one end system. How does it instruct the internet to deliver data to another program running on another end system?

Through the use of socket interfaces, which specifies how a program running on one end system asks the Internet infrastructure to deliver data to a specific destination program running on another end system.

As an analogy - when sending a letter, you need to put a letter in an envelope, write the full name and address and zip code on the envelope, seal the envelope, put a stamp on the envelope in a specific location, and drop the envelope into a mailbox.

What is a Protocol?

A network protocol is a way for computers to talk to each other - it defines the format and the order of messages exchanged between two communication entities, as well as the actions taken on the transmission and/or receipt of a message

As an analogy, take asking a person for time. You say hi, the other person says hi back, you ask for the time, they reply the time (1:23 PM as of this writing).

1.2 The Network Edge

The ‘edge’ of the network refers to the systems sitting at the edge of the internet - the end systems/hosts that are connected to the internet

They are called end systems because they sit at the edge of the internet. These are computers and laptops, and mobile devices, and things that are connected to the internet but really shouldn’t be (IoT).

Hosts can be divided into clients and servers. Clients are desktops, laptops, smartphones, etc. Servers store and distribute web pages, stream video, relay e-mail, etc. Today, most of the servers are in data centers.

The access network - the network that physically connects an end system to the first router on a path from the end system to any other distant end system.

The Network Core

The network core is the mesh of packet switches and links that interconnect the internet’s end systems

Packet switching - the hosts break application-layer messages into packets - routers and link-layer switches.

Transmitted over each communication link at a rate equal to the full transmission rate of the link. A packet of L bits with a transmission rate R bits/sec, the time to transmit the packet is L/R seconds.

Forwarding - a local action, all about moving arriving packet from router’s input link to appropriate router output link

Routing - a global action to determine source-destination taken by packets

As an analogy:

Forwarding: Taking a long trip across the country - New York to Los Angeles - you will pass through cities (input) and need to get out of the city (output). The route from NY to LA is the routing.

Packet transmission delay: if a packet is L bits long, and the transmission rate is R bits/sec, then it will take L/R seconds to transmit (push out) L-bit packet into link at R bps

Store and forward - entire packet must arrive at router before it can be transmitted onto next link

With a network that consists of a source, a router, and a destination:

Source sends packets of L bits at a transmission rate of R bits/sec.

At time 0, the source starts sending the first packet.

At time L/R seconds, the full packet arrives at the router.

The router must wait until it gets the full packet before sending it out.

At time 2L/R, the full packet reaches the destination.

Sending one packet from one source to destination over a path of N links each at rate R, the end-to-end delay is N (L/R)

Queueing occurs when work arrives faster than it can be serviced

Packet queues arrive at the router’s outbound link - if arrival rate in bps to link exceeds transmission rate (bps) of link for some period of time

Packets can be dropped if memory in router fills up (lost, buffer)

Packet Switching - messages are exchanged with end systems over network applications. Sending a message from a source end system to a destination end system, the source breaks long messages into smaller chunks of data known as packets. Packets travel through communication links and packet switches (routers and link-layer switches). The rate is equal to the full transmission rate of the link. L bits over a link with transmission rate R bits/sec, the time to transmit the packet is L/R

Messages can be anything the application designer wants.

Store-And-Forward transmission: The packet switch must receive the entire packet before it can begin to transmit the first bit of the packet onto the outbound link. Only after all the bits have been received can the router transmit (”forward”) the packet onto the outbound link.

If it takes L/R seconds to send one full packet, the total time for one packet to get from the source to the destination through the router is 2L/R.

  1. L/R seconds for the source to send the packet to the router.
  1. Another L/R seconds for the router to send it to the destination.

If the router forwarded them as soon as they arrived, then it would just be L/R.

The general case of sending one packet from source to destination over a path consisting of N links each of rate R. The end-to-end delay is N (L/R).

Each packet switch has multiple links attached to it - for each attached link, the packet switch has an output bufer/output queue that stores packets that the router is about to send into that link.

The amount of buffer space is finite. Arriving packets may be dropped (called packet loss) if the buffer is full

Each router has a forwarding table that maps destination addresses (or portions of the destination addresses ) to that router’s outbound links. When a packet arrives at a router, the router examines the address and searches its forwarding table, using this destination address, to find the appropriate outbound link. The router then directs the packet to this outbound link.

In the internet, every end system has an address called an IP address. When a source wants to send a packet to a destination, the source includes the destination’s IP address in the packet’s header. When a packet arrives at arouter in the network, the router examines a portion of the packet’s destination address and forwards the packet to an adjacent router. The router has a forwarding table that maps destination addresses to that router’s outbound links.

An analogy of a car driver who prefers to ask for directions instead of using maps.

The internet has a number of special routing protocols that are used to automatically set the forwarding tables.

Circuit Switching - the resources needed along a path (buffers, link transmission rate) to provide for communication between the end systems are reserved for the duration of the communication session.

Before a communication occurs, the end-end resources are allocated

Circuits aren’t shared between users

Think of the difference between a restaurant that needs reservations versus one that does not

A circuit in a link is implemented with Frequency Division Multiplexing or Time Division Multiplexing

FDM - The frequency spectrum of a link is divided up among the connections established across the link

The electromagnetic frequencies are divided into narrow frequency bands

Each call allocated its own band

TDM - time is divided into frames of fixed duration, and each frame is divided into a fixed number of time slots.

time is divided into slots

each call allocated periodic slots, can transmit at maximum rate of (wider) frequency band (only) during its time slots

Dedicated circuits are idle if there is no communication happening, and no other users can use that circuit.

Potentially wasteful.

A Network of Networks

You get internet access from an ISP, and ISPs must be interconnected - called a network of networks.

Building a series of networking structures:

The overarching goal is to interconnect the access ISPs, so all end systems can communicate with each other.

One naive approach is to connect every access ISP directly. It is extremely costly, and would require each access ISP to have a separate communication link to each of the hundreds of thousands of other access ISPs all over the world.

Network Structure 1 - interconnects all the access ISP with a single global transit ISP. It spans the globe and also has at least one router near each of the hundreds of thousnads of access ISPs. If some company builds this, then it is natural for other companies to build their own, which leads us to the next one.

Network Structure 2 - hundreds of thousands of access ISPs and multiple global transit ISPs. Two-tiered hierarchy with global transit providers resiing at the top tier and access ISPs at the bottom tier.

Assumes that global transit ISPs are not only capable of getting close to each and every access ISP, but also find it economically desirable to do so.

In reality, no ISP has presence in each and every city in the world. In any given region, there is a regional ISP to which the access ISPs in the region connect. Each regional ISP then connects to tier-1 ISPs.

There may be multiple competing regional ISPs in a region. Each access ISP pays the regional ISP and each regional ISP pays the tier-1 ISP to which it connects.

Adding points of presence - a group of one or more routers in the provider’s network where customer ISPs can connect into the provider ISP.

Multi-home - connect to two or more provider ISPs.

Internet Exchange Point - a meeting point where multiple ISPs can peer together.

Content-provider networks - companies like Google

Delay, Loss, and Throughput In Packet-Switched Networks

Throughput - the amount of data per second that can be transferred

As a packet starts in a host (the source), passes through a number of nodes (host or routers) to the next node, the packet suffers from several types of delays at each node along the path.

When a packet goes through Router A to Router B, Router A must examine the packet’s head to decide the correct outbound link. The packet can only be transmitted if the link is idle and no other packets are in the queue. If the link is busy or other packets are queued, then the new packet must wait in the queue.

Processing Delay - the time required to examine the packet’s header and determine where to direct the packet.

The time it takes to actually process a packet as it gets to the router

Microseconds in practice

Queuing Delay - The packet is waiting to be transmitted on the link

Can be 0 if there is no queue, if traffic is heavy, can be on the order of microseconds to milliseconds

Transmission Delay - The amount of time required to push (transmit) all of the packet’s bits into the link

Length of packet by L bits, transmission rate from Router A to router B by R bits/sec. For a 10 Mbps Ethernet link, the rate is R= 10 Mbps. The transmission delay is L/R.

Propagation Delay - Once a bit is pushed into the link, it needs to propagate (transmit) to router B. Propagation delay is the time required to propagate from the beginning of the link to router B. Depends on the physical medium of the link (fiber optics, twisted-pair copper wire, etc) and is in the range of :

2 * 10^8 meters/sec to 3 * 10^8 meters/sec

Equal to or less than the speed of light

The distance between two routers divided by the propagation speed

d/s

The difference between transmission delay and propagation delay:

transmission delay is the time it takes for a router to “push out” the entire packet onto the link. It depends on the packet length and transmission rate but not the physical distance.

Propagation Delay is the time it takes for a bit to physically travel from one router to the next over the link. It depends on the distance between routers and the speed of signal propagation, not the packet length.

Total node delay is equal to the processing, queuing, transmission, and propagation delays combined.

Queuing Delay and Packet Loss

Unlike processing, transmission, and propagation delay, queueing delay can vary from packet to packet. If 10 packets arrive at an empty buffer, the first packet will experience 0 delay, while the last packet will experience the most delay.

When characterizing queuing delay, one typically uses statistical measures such as average queuing delay.

L = packet length in bits

a = average rate at which packets arrive (units of packets/sec)

R = transmission rate.

Average rate at which bits arrive at the queue is La bits/sec. La/R is the traffic intensity. if La/R > 1, the average rate at which bits arrive at the queue exceeds the rate at which bits can be transmitted from the queue.

Packet Loss - A queue preceding a link has a finite capacity for packets. Packet delays do not really approach infinity because of that finite capacity. If there is no place to store the packets, then the packet is dropped - the packet is loss.

Total Delay from source to destination - end to end delay

Suppose there are N - 1 routers with minimal network congestion. The total delay will be N(processing delay + transmission delay + propagation delay)

Instantaneous: rate at given point in bits/sec - real-time data transfer

Average: rate over longer period of time - calculated as Total file size F divided by transfer time T

Protocol Layers and Their Service Models

Network designers organize protocols in layers. Each protocol belongs to one of the layers.

A protocol layer can be implemented in software, hardware or a combination of the two.

Physical and data link layers are responsible for handling communication over a given link.

The Internet Protocol Stack

Application - where network applications and their application-layer protocols reside

HTTP, SMTP, FTP

An application layer protocol is distributed over multiple end systems

Packet of information is called a message

Transport - transports application-layer messages between application endpoints

TCP - provides a connection-oriented service to its applications - guaranteed delivery

UDP - connectionless service - no reliability, no flow, no congestion control

Packet of information is called a segment

Network Layer - responsbile for moving network-layer packets known as datagrams from one host to another. Responsible for the actual routing.

IP protocol

Link Layer - Moves the datagrams from one node (host or router) to the next node in the router.

Packet of information is called a frame

Physical Layer - the job of the physical layer is to move individual bits within a frame from one node to the next. The protocols in this layer are link dependent - Ethernet has many different protocols for twisted-pair copper wire, coaxial cable, etc.

Encapsulation

Each layer of the protocol stack adds its own header information to the data before passing it to the next layer.

Encapsulation happens everywhere in the internet

At the sending host, an application layer message is passed to the transport layer. The transport layer adds its own information to form a transport-layer segments, and then pass it to the network layer. The networ-layer adds its own information to form a datagram. Then it passes it to the link layer which adds its own information to form a a frame.

Networks Under Attack

The internet is critical for many institutions today.

Bad Guys attempt to attack computer networks.

The Bad Guys Can Put Malware into Your Host Via the Internet

Malware - malicious software that can infect hosts and do a lot of things that are bad like deleting files, installing spyware to collect private information. Compromised hosts may be enrolled in a botnet, which is where bad guys control and leverage the host for further terrible things.

Malware is self replicating - once it infects one host, from that host it seeks entry to other hosts over the Internet.

The Bad Guys Can Attack Servers and Network Infrastructure

A denial-of-service attack is when a network is intentionally overwhelmed with traffic, with the goal of shutting down the network. These can fall into three categories:

  1. Vulnerability Attack - a few well-crafted messages are sent to a vulnerable application or operating system running on the target host. If there is the right sequence of packets, then the service can stop/host can crash.
  1. Bandwidth flooding - the attacker sends a deluge of packets to the targeted host - so many packets that the target’s access link becomes clogged
  1. Connection flooding - the attacker establishes a large number of half-open or fully open TCP connections at the target host, in order to bog down the host with these connections.

If a server has an acess rate of R bps, then there will need to be traffic at approximately R bps to cause damage. Attackers can utilize a botnet to conduct a distributed Denial of Service attack to cause that traffic.

The Bad Guys Can Sniff Packets

A packet sniffer is a passive receiver that records a copy of every packet that flies by.

The Bad Guys Can Masquerade as Someone You Trust

IP Spoofing - faking the IP address for someone the target would trust.

The Internet was originally designed with the model of “a group of mutually trusting users attached to a transparent network.” By definition, that makes it insecure.
