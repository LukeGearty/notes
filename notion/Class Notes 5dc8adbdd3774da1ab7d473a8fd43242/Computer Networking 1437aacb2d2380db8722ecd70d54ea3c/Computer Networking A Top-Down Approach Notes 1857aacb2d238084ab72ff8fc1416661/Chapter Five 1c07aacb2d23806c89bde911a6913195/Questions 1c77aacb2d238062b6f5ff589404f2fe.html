Section 5.1



R1.  What is meant by a control plane that is based on per-router control? In such cases, when we say the network control and data planes are implemented “monolithically,” what do we mean?


An algorithm runs in each router. Forwarding (data plane) and routing (control plane) is contained within the router.



R2.  What is meant by a control plane that is based on logically centralized control? In such cases, are the data plane and the control plane implemented within the same device or in separate devices? Explain.

A logically centralized controller is responsible for the control plane functions of the router. That is implemented separately from the routers, which only implement the data plane functions.

Section 5.2




R3.  Compare and contrast the properties of a centralized and a distributed routing algorithm. Give an example of a routing protocol that takes a centralized and a decentralized approach.


A centralized routing algorithm comptues the least cost path between source and destination using complete knowledge of the network. A distributed routing algorithm is run by routers in an iterative, distributed manner. The routers do not have complete topology, and only know the direct costs of its neighbors. Through getting information in an iterative way, it gradually calculates the least cost path.

Centralized: OSPF

Decentralized: BGP

R5.  What is the “count to infinity” problem in distance vector routing?

In distance vector routing algorithm, the “count to infinity” problem occurs when there is “bad news”, that is when a link costs go up. The time it will take to propagate this link cost increase through the network is rather high.



R6.  Is it necessary that every autonomous system use the same intra-AS routing
algorithm? Why or why not?

No, because each AS has different needs and requirements based on topology, traffic patterns and policies, and it needs to be able to operate with different ASs.

Because the different ASes are controlled by different ISPs who have different policies for their networks. Also the scale of the internet is too large.

Section 5.5



R16.  What types of messages flow across an SDN controller’s northbound and southbound APIs? Who is the recipient of these messages sent from the controller across the southbound interface, and who sends messages to the controller across the northbound interface?

Northbound: The controller is interacting with network-control applications. It allows the applications to read/write network state and flow tables.

Southbound: The SDN controller is interacting with controlled network devices. These messages provide the controller with an updated view of the network’s state. These messages can indicate the status of a link, a device has just gone down, a device is operational.



R17.  Describe the purpose of two types of OpenFlow messages (of your choosing) that are sent from a controlled device to the controller. Describe the purpose of two types of Openflow messages (of your choosing) that are send from the controller to a controlled device.

Controlled Switch to controller:

Flow-Removed: informs the controller that a flow table entry has been removed

Port Status: used by the switch to inform the controller of a change in port status

Controller to Controlled Switch:

Configuation: queries and sets a switch’s configuration parameters.

Modify-State: message used by a controller to add/delete or modify entries in the switch’s flow table and set switch port properties.
