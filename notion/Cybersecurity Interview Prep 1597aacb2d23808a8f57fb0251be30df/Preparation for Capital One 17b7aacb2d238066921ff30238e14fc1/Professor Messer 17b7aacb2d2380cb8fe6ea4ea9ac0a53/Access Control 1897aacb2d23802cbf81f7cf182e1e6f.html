Access Control

Once someone authenticates to a network, we need to provide them resources they need to perform their job

Least Privilege - rights and permissions should be set to the bare minimum

You only get what you need to complete your objective

All user accounts must be limited

If someone runs malicious software it just affects that system

Mandatory Access Control - The operating system limits the operation on an object

Assigns a label to each object gets a label (confidential, secret, top secret, etc)

The administrator decides who gets access to what security level

Users cannot change these settings

Discretionary Access Control - used in most operating systems - a familiar access control model

You create a spreadsheet, you control who has access

You can modify access

Very flexible, it is weaker security than MAC though

Role-based Access Control - RBAC - access control is based on the job function/role in the organization

Administrators provide access based on the role of the user

Rights are gained implicitly instead of explicitly

Once group is created, users are added to group

In Windows, use Groups to provide role-based access control

Rule-based Access Control

Generic term for following rules

Conditions other than who you are

Access is determined through system-enforced rules

Attribute-based Access Control (ABAC) - with ABAC, there are many different criteria to determine access

Complex rules can be created

A “next generation” authorization model

Time-of-day restrictions - almost all security devices include a time-of-day option

Restrict access during certain times or days of the week

Usually not the only access control

Can be difficult to implement in a 24-hour environment

Notes: Access Control Models and Methods

1. Access Control Overview

  • Definition: Process of enforcing policies that allow or disallow access to data.
  • Purpose: Provide authenticated users access to resources needed for their job functions.
  • Implementation:
    • Policies define access requirements.
    • IT teams translate policies into system-enforced processes.

2. Security Best Practice: Least Privilege

  • Definition: Assign users only the rights and permissions required for their job.
  • Key Features:
    • Limits default privileges to reduce potential damage from malicious software.
    • No administrator access unless explicitly needed.

3. Access Control Models

A. Mandatory Access Control (MAC)

  • Key Features:
    • Resources are labeled (e.g., confidential, secret, top secret).
    • Administrators define user permissions based on labels.
  • Use Case: Highly secure environments (e.g., government, military).

B. Discretionary Access Control (DAC)

  • Key Features:
    • Data creators control access and permissions for their data.
    • Example: Creator of a spreadsheet decides who can read or modify it.
  • Pros:
    • Flexibility for data owners.
  • Cons:
    • Less secure; relies on individual users to set appropriate controls.

C. Role-Based Access Control (RBAC)

  • Key Features:
    • Permissions assigned to roles based on job functions (e.g., manager, director).
    • Administrators define groups and assign users to those groups.
    • Example: Shipping department group may have permissions for shipping software.
  • Pros:
    • Centralized control.
    • Easier management of permissions.

D. Rule-Based Access Control

  • Key Features:
    • System-enforced rules determine access.
    • Rules created by administrators, not users.
    • Example Rules:
      • Access allowed only between 9:00 AM and 5:00 PM.
      • Forms can only be filled using a specific browser (e.g., Chrome).

E. Attribute-Based Access Control (ABAC)

  • Key Features:
    • Uses multiple criteria to determine access (e.g., IP address, time, user action).
    • Combines rules for more complex access control.
    • Example: Access depends on IP, time of day, and relationship to data.
  • Pros:
    • Flexible and granular.
    • Suitable for complex authorization requirements.

4. Time-of-Day Restrictions

  • Definition: Restricts access based on time.
  • Examples:
    • Training room network inaccessible from midnight to 6:00 AM.
    • R&D databases accessible only from 8:00 AM to 6:00 PM.
  • Challenges:
    • Managing time zones in global organizations.
