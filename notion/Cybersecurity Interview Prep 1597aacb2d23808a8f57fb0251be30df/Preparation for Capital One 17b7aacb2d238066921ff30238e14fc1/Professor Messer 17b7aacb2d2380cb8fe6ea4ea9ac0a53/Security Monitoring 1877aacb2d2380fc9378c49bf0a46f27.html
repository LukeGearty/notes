Actively check systems and devices.

We look for reports that can give us action items - status information on what devices need to be updated (software, hardware, OSes).

Importance of Network Monitoring

  • Attackers constantly target systems and services, necessitating continuous monitoring.
  • Key areas to monitor:
    • Authentications and logins: Verify legitimate access.
    • Services and remote access: Identify suspicious activity.
  • Real-time monitoring allows quick reactions to security events:
    • Account access
    • Firewall rule base
    • Additional scanning for abnormal activities

Monitoring Tools and Dashboards

  • Dashboards provide a high-level view of security posture.
  • Key monitoring points:
    • Computing resources: Identify unusual authentications (e.g., logins from unexpected countries).
    • Services: Check activity levels, backups, and software versions.
    • Applications: Monitor availability and operational status.
  • Traffic monitoring can uncover data exfiltration attempts (e.g., large, unusual data transfers).

Consolidation with SIEM

  • Security Information and Event Manager (SIEM):
    • Centralizes logs from various devices (firewalls, switches, servers, routers).
    • Simplifies reporting and correlation of diverse data.
    • Enables actionable insights, such as VPN authentication tracking or data transfer analysis.

Vulnerability Scanning

  • Systems are in constant motion (e.g., laptops, mobile devices), making tracking vulnerabilities challenging.
  • Scanning software gathers data on:
    • Operating systems and versions
    • Device drivers
    • Applications
    • Anomalies logged in devices
  • Reports from scans are used to:
    • Identify non-compliance with patches.
    • Plan remediation actions for vulnerable systems.
    • Analyze future risks (e.g., end-of-life systems).

Long-Term Threat Detection

  • Breaches often go unnoticed for months (e.g., IBM study: average detection time is nine months).
  • Long-term backups are essential to address undetected breaches.
  • Legal requirements may mandate extended data retention for security analysis.

Alerts and Alarms

  • Real-time alerts notify of unusual events:
    • Increased authentication errors (e.g., brute-force attacks).
    • Sudden spikes in data transfers.
  • Notifications can be delivered via text, email, or SOC-specific systems.
  • Immediate actions include quarantining compromised systems to limit attacker movement.

Challenges in Alert Management

  • False Positives: Alerts for non-issues; require tuning to reduce noise.
  • False Negatives: Missed critical events; need adjustments to avoid gaps.
  • Continuous refinement ensures accurate alerts for timely and effective responses.

Summary

  • Effective monitoring involves a combination of real-time detection, centralized logging (via SIEM), actionable reporting, and proactive alerting systems.
  • Ongoing challenges include tuning alerts, managing false positives/negatives, and maintaining long-term security strategies.
