NMAP related Questions

  1. What is Nmap?
    • Answer: Nmap (Network Mapper) is a free and open-source tool used for network discovery and security auditing. It can scan large networks and provide information about live hosts, services running on ports, operating systems, and vulnerabilities.
  1. How does Nmap perform host discovery?
    • Answer: Nmap uses various techniques for host discovery, including:
      • Ping scans: Sends ICMP Echo Requests to check if hosts are online.
      • ARP scans: Used on local networks to identify hosts via Address Resolution Protocol.
      • TCP and UDP scans: Sends packets to specific ports to determine if a host is responsive.
  1. What are the common types of scans in Nmap?
    • Answer:
      • TCP Connect Scan (-sT): Completes a full TCP three-way handshake.
      • SYN Scan (-sS): Sends SYN packets to identify open ports without completing the handshake.
      • UDP Scan (-sU): Probes UDP ports to identify open services.
      • Ping Scan (-sn): Discovers live hosts without port scanning.
  1. What does the output open, closed, and filtered mean in Nmap?
    • Answer:
      • Open: The port is actively accepting connections.
      • Closed: The port is not open but accessible.
      • Filtered: Nmap cannot determine whether the port is open due to firewalls or other filtering mechanisms.

  1. What is Nmap, and why is it used?
    • Answer: Nmap (Network Mapper) is an open-source tool for network discovery and security auditing. It is used to scan networks, identify hosts, services, open ports, operating systems, and potential vulnerabilities.
  1. How do you perform a basic scan with Nmap?
    • Answer: Use the command:This performs a default scan, checking 1,000 commonly used TCP ports on the specified target.
      nmap <target>
  1. What is the difference between TCP Connect Scan and SYN Scan?
    • Answer:
      • TCP Connect Scan (-sT): Completes the full three-way handshake, making it more detectable but reliable.
      • SYN Scan (-sS): Sends a SYN packet and waits for a response (half-open scan). It is faster and stealthier than a TCP Connect Scan.
  1. What is the purpose of the A option in Nmap?
    • Answer: The A option enables aggressive scanning, which includes OS detection, version detection, script scanning, and traceroute.
  1. How do you save Nmap results to a file?
    • Answer: Use the oN, oX, or oG options:
      • oN: Save as a normal text file.
      • oX: Save as an XML file.
      • oG: Save as a greppable file for scripting.

Intermediate-Level Questions

  1. Explain how Nmap identifies open ports.
    • Answer: Nmap sends probes to target ports and analyzes the responses. If it receives a SYN/ACK, the port is open. If it receives a RST, the port is closed. If there’s no response, the port may be filtered.
  1. What is a UDP scan in Nmap, and how is it different from a TCP scan?
    • Answer: A UDP scan (sU) sends UDP packets to target ports. Unlike TCP scans, it does not rely on a handshake. Responses like ICMP Port Unreachable indicate closed ports, while no response or application-specific replies suggest open/filtered ports.
  1. What are NSE scripts, and how are they used in Nmap?
    • Answer: NSE (Nmap Scripting Engine) scripts allow users to extend Nmap's capabilities to perform tasks like vulnerability detection, brute force attacks, and service enumeration. Use them with:
      bash
Copy code
nmap --script <script_name> <target>
  1. How would you perform an OS detection scan?
    • Answer: Use the O option:
      bash
Copy code
nmap -O <target>
  1. What is the p option used for in Nmap?
    • Answer: The p option specifies the ports to scan. For example:
      • p80: Scan port 80 only.
      • p1-1000: Scan ports 1 to 1000.
      • p-: Scan all 65,535 ports.

Advanced-Level Questions

  1. How can Nmap bypass firewalls or IDS/IPS systems?
    • Answer: Techniques include:
      • Fragmentation: -mtu <value> breaks packets into smaller fragments.
      • Decoy Scans: D RND:10 uses decoy IPs to obscure the source.
      • Timing Options: Using T0 or T1 for stealthy scans.
      • Source Port Manipulation: -source-port <port> can use allowed ports like 80 or 443.
  1. How do you perform a scan of multiple IP addresses or a subnet?
    • Answer: Use:
      • A range: nmap 192.168.1.1-254
      • A CIDR notation: nmap 192.168.1.0/24
      • Multiple targets: nmap 192.168.1.1 192.168.1.2
  1. What is a stealth scan, and why is it useful?
    • Answer: A stealth scan (e.g., SYN scan) sends minimal packets to avoid detection by IDS/IPS systems. It doesn't complete the TCP handshake, making it harder to log.
  1. What is the purpose of the -reason option in Nmap?
    • Answer: The -reason option provides detailed explanations for why Nmap considers a port to be open, closed, or filtered.
  1. How would you perform a scan to detect hosts behind a firewall?
    • Answer: Use idle scans (sI) or use NSE scripts designed for detecting firewalls or proxies.

Scenario-Based Questions

  1. You scanned a host, and all ports are marked as filtered. What does this indicate?
    • Answer: It indicates that a firewall or packet filtering system is blocking the probes. No response is received from the target.
  1. You need to enumerate services and their versions on a target. Which options do you use?
    • Answer: Use the service/version detection option:
      bash
Copy code
nmap -sV <target>
  1. How can you speed up an Nmap scan?
    • Answer:
      • Use T4 or T5 timing templates.
      • Limit the number of ports scanned with p.
      • Use fast scan mode with F.
      • Avoid heavy options like A or OS detection (O).
  1. How do you check if a specific vulnerability exists on a target using Nmap?
    • Answer: Use NSE vulnerability scripts, for example:
      bash
Copy code
nmap --script vuln <target>
  1. What is the output of the command nmap -sS -O -T4 -p 1-1000 192.168.1.1?
    • Answer: It performs a SYN scan on ports 1 to 1000 of 192.168.1.1, detects the OS, uses timing template T4 for faster scans, and provides detailed results.
