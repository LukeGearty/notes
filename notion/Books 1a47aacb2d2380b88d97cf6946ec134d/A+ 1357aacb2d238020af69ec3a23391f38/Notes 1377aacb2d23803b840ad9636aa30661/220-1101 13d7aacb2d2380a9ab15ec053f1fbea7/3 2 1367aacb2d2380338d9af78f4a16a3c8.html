  • Parity Checking: A basic error detection method used in computer memory where an extra bit, called the parity bit, is added to each byte of data to check for errors. If the number of bits with a value of "1" is odd (odd parity) or even (even parity), the parity bit helps verify data accuracy, but it can't correct errors.
  • Error-Correction Code (ECC): ECC memory adds extra bits to detect and correct errors automatically, beyond just detecting them. ECC can correct single-bit errors and detect double-bit errors, which makes it ideal for systems where data integrity is critical, like servers.
  • Single-Sided and Double-Sided Memory: These terms refer to how the memory chips are arranged on the memory module. Single-sided memory has chips on only one side, while double-sided memory has chips on both sides, effectively allowing for more storage capacity per module.
  • Single, Dual, Triple, and Quad-Channel Memory: These refer to the memory controller’s ability to access multiple memory modules simultaneously to increase bandwidth.
    • Single-Channel: Accesses one memory module at a time, providing the least bandwidth.
    • Dual-Channel: Accesses two modules simultaneously, doubling bandwidth.
    • Triple-Channel: Accesses three modules at once, offering even more bandwidth (rarely used in most modern PCs).
    • Quad-Channel: Accesses four modules simultaneously for maximum bandwidth, common in high-performance computing setups.

DRAM (Dynamic Random-Access Memory) is a type of memory used in computers and other devices to store data that the processor needs quickly. It is called "dynamic" because it stores each bit of data in a tiny capacitor within an integrated circuit, which needs to be refreshed thousands of times per second to maintain the data.

Key characteristics of DRAM:

  • Volatile Memory: Data is lost when power is turned off.
  • High Density: Can store a large amount of data per chip, making it cost-effective.
  • Fast Access: Used in main memory (RAM) for quick access by the CPU, but it’s slower than the CPU cache.

Asynchronous DRAM (ADRAM) is a type of DRAM that operates independently of the system clock, meaning it doesn’t synchronize with the CPU’s clock speed. Instead, it uses control signals to manage data flow, which can make it slower and less efficient in high-speed operations compared to synchronous DRAM (SDRAM), which does operate in sync with the CPU clock.

In ADRAM, memory access times depend solely on the DRAM's internal timing, not on an external clock. This makes it simpler but slower for modern systems where high-speed synchronization with the CPU is needed. Due to these limitations, asynchronous DRAM is now largely outdated and has been replaced by faster, synchronous types like SDRAM and DDR RAM in most systems.

Synchronous DRAM (SDRAM) is a type of DRAM that is synchronized with the system clock, allowing it to operate in sync with the CPU’s clock speed. This synchronization enables SDRAM to be much faster and more efficient than asynchronous DRAM because it can anticipate when the CPU will request data, allowing it to prepare data for quicker access.

Key characteristics of SDRAM:

  • Clock Synchronization: SDRAM uses the system clock to time its operations, reducing delays and increasing data throughput.
  • Pipelining: SDRAM can queue multiple instructions at once, which means it can start fetching the next piece of data before the previous operation is complete, further speeding up performance.
  • Foundation for Modern RAM: SDRAM led to the development of later generations, like DDR (Double Data Rate) SDRAM, which provides even higher speeds and bandwidth.

Because of its speed and efficiency, SDRAM and its derivatives are the standard types of memory used in modern computers and devices.
