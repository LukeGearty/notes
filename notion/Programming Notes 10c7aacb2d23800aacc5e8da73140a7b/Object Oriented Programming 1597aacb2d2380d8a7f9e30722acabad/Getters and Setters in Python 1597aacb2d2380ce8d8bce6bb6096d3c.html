Getters and Setters in Python

Notes on Getters and Setters in Python

What Are Getters and Setters?

  • Getters and setters are methods used to retrieve and update the value of a property in an object.
  • They help ensure controlled access to an object’s attributes, often adding validation or other logic when accessing or modifying the attributes.

Why Use Getters and Setters?

  1. Encapsulation:
    • Helps protect internal state by preventing direct access to attributes.
  1. Validation:
    • Allows for adding checks when setting or getting a value.
  1. Abstraction:
    • Hides implementation details while providing controlled access.
  1. Flexibility:
    • You can change the implementation later without affecting external code.

Traditional Way of Using Getters and Setters

  • Before the @property decorator, getters and setters were implemented as methods.
python
Copy code
class Person:
    def __init__(self, name):
        self._name = name  # Use a private or protected attribute

    def get_name(self):
        return self._name

    def set_name(self, value):
        if not isinstance(value, str):
            raise ValueError("Name must be a string!")
        self._name = value

# Example usage
person = Person("Alice")
print(person.get_name())  # Get the name
person.set_name("Bob")    # Set the name

Modern Approach: Using @property Decorator

  • Python provides the @property decorator to simplify the creation of getters and setters.
python
Copy code
class Person:
    def __init__(self, name):
        self._name = name

    @property
    def name(self):
        """Getter for name"""
        return self._name

    @name.setter
    def name(self, value):
        """Setter for name"""
        if not isinstance(value, str):
            raise ValueError("Name must be a string!")
        self._name = value

# Example usage
person = Person("Alice")
print(person.name)  # Accesses the getter
person.name = "Bob"  # Calls the setter

Key Points About @property:

  1. @property:
    • Converts a method into a getter.
    • Allows you to access the attribute like a regular property.
  1. @<property>.setter:
    • Allows defining a setter for the same property.
  1. @<property>.deleter (Optional):
    • Allows adding logic for deleting a property.

Advantages of @property:

  • Cleaner and more Pythonic syntax.
  • Backward compatibility with existing code.
  • Avoids breaking the interface if a property transitions from public to private.

Example with Full Property Control

python
Copy code
class Rectangle:
    def __init__(self, width, height):
        self._width = width
        self._height = height

    @property
    def width(self):
        return self._width

    @width.setter
    def width(self, value):
        if value <= 0:
            raise ValueError("Width must be positive!")
        self._width = value

    @property
    def height(self):
        return self._height

    @height.setter
    def height(self, value):
        if value <= 0:
            raise ValueError("Height must be positive!")
        self._height = value

    @property
    def area(self):
        """Read-only property for area"""
        return self._width * self._height

# Example usage
rect = Rectangle(5, 10)
print(rect.area)  # 50 (read-only property)
rect.width = 7
print(rect.area)  # 70

Use Cases for Getters and Setters

  • Data validation (e.g., ensuring positive values for dimensions).
  • Lazy evaluation (computing a value only when accessed).
  • Maintaining consistency (updating dependent values when one changes).
  • Read-only properties (e.g., calculated attributes like area).

Conclusion

  • Use getters and setters when control or validation is needed for attribute access.
  • Use @property for a clean and Pythonic implementation.
