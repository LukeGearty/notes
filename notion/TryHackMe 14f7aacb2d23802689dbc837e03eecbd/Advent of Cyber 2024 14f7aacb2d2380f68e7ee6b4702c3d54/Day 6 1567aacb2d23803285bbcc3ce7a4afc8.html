4. Detecting Sandboxes

Understanding Sandboxes

  • Definition:
    • Isolated environments where code is executed without impacting the host system.
    • Equipped with tools for monitoring, recording, and analyzing code behavior.

Malware's Sandbox Detection Technique

  • Registry Check:
    • Queries the Registry path: HKLM\Software\Microsoft\Windows\CurrentVersion
    • Looks for the presence of the C:\Program Files directory.
    • Absence may indicate a sandbox or virtualized environment.

C Code Implementation

c
Copy code
void registryCheck() {
    const char *registryPath = "HKLM\\Software\\Microsoft\\Windows\\CurrentVersion";
    const char *valueName = "ProgramFilesDir";

    // Prepare the command string for reg.exe
    char command[512];
    snprintf(command, sizeof(command), "reg query \"%s\" /v %s", registryPath, valueName);
    // Run the command
    int result = system(command);
    // Check for successful execution
    if (result == 0) {
        printf("Registry query executed successfully.\n");
    } else {
        fprintf(stderr, "Failed to execute registry query.\n");
    }
}
int main() {
    const char *flag = "[REDACTED]";
    registryCheck();
    return 0;
}
  • Explanation:
    • Checks for ProgramFilesDir in the registry to determine the environment.
    • Helps malware decide whether to proceed with malicious activities.

5. YARA Rules for Malware Detection

Introduction to YARA

  • Purpose:
    • Identify and classify malware based on code patterns.
  • Functionality:
    • Analysts create custom rules defining specific characteristics (strings, headers, behaviors).
    • YARA scans files/processes for matches to these rules.

YARA Rule Example

yara
Copy code
rule SANDBOXDETECTED
{
    meta:
        description = "Detects the sandbox by querying the registry key for Program Path"
        author = "TryHackMe"
        date = "2024-10-08"
        version = "1.1"

    strings:
        $cmd = "Software\\Microsoft\\Windows\\CurrentVersion\" /v ProgramFilesDir" nocase

    condition:
        $cmd
}
  • Components:
    • Meta: Descriptive information about the rule.
    • Strings: Defines patterns to search for (e.g., specific registry queries).
    • Condition: Specifies when the rule should trigger (presence of $cmd).

YARA Rule Application

  • Testing:
    • Mayor Malware sets up a script to execute the YARA rule on new Sysmon events.
    • Logs true positives in C:\Tools\YaraMatches.txt.

8. Utilizing Floss for String Extraction

Understanding Floss

  • Function:
    • Extracts obfuscated or hidden strings from malware binaries.
  • Advantages:
    • More effective than traditional strings tool for malware analysis.

Running Floss

  • Command:
    powershell
Copy code
PS C:\Tools\FLOSS> floss.exe C:\Tools\Malware\MerryChristmas.exe | Out-File C:\tools\malstrings.txt
  • Process:
    1. floss.exe scans MerryChristmas.exe for embedded strings.
    1. Output Redirection:
      • | pipes the output to Out-File, saving results to malstrings.txt.
    1. Post-Processing:
      • Open malstrings.txt.
      • Search for "Mayor Malware" to find the flag (THM{...}).

Command Breakdown

  • floss.exe C:\Tools\Malware\MerryChristmas.exe:
    • Scans the specified executable for hidden strings.
  • | (Pipe):
    • Directs the output of floss.exe to the next command.
  • Out-File C:\tools\malstrings.txt:
    • Saves the extracted strings to malstrings.txt.

9. Applying YARA Rules on Sysmon Logs

Challenges for Malware

  • YARA rules are effective against Mayor Malware's evasion techniques.

Sysmon Overview

  • Tool: Part of Microsoft's Sysinternals suite.
  • Function:
    • Monitors and logs system activity (process creation, network connections, file changes).
    • Provides detailed event data useful for tracing malware behavior.

Custom YARA Rule Application

  • Objective:
    • Detect malware activities by scanning Sysmon logs for specific event IDs.

Procedure

  1. Retrieve YARA Match Logs:
    powershell
Copy code
PS C:\Tools> get-content C:\Tools\YaraMatches.txt
    • Example Output:
      yaml
Copy code
Event Time: 10/11/2024 15:06:39
Event ID: 1
Event Record ID: 96517
Command Line: reg query "HKLM\Software\Microsoft\Windows\CurrentVersion" /v ProgramFilesDir
YARA Result: DetectShutdownTimeQuery C:\Users\Administrator\AppData\Local\Temp\2\tmp8D61.tmp
  1. Note Event Record ID:
    • Example: 96517
  1. Filter Sysmon Logs in Event Viewer:
    • Navigate:
      • Open Windows Event Viewer.
      • Go to Applications and Services Logs → Microsoft → Windows → Sysmon → Operational.
    • Apply Filter:
      1. Click on Filter Current Log.
      1. Select the XML tab.
      1. Check Edit query manually and confirm.
      1. Insert the custom filter:
        xml
Copy code
<QueryList>
  <Query Id="0" Path="Microsoft-Windows-Sysmon/Operational">
    <Select Path="Microsoft-Windows-Sysmon/Operational">
      *[System[(EventRecordID="96517")]]
    </Select>
  </Query>
</QueryList>
      1. Replace 96517 with the noted Event Record ID.
      1. Click OK to apply the filter.
  1. Analyze Event Details:
    • Details Tab:
      • ParentImage: Identifies the parent process (MerryChristmas.exe).
      • ParentProcessId & ProcessId: Useful for tracking related events.
      • User: Determines privileges used (e.g., hidden accounts).
      • CommandLine: Details of the executed command.
      • UtcTime: Time frame for the malware's operation.
